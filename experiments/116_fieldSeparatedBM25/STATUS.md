# 116: Field-Separated BM25 — 실험 완료

## 목적

현재 `dartlab.search()`가 report_nm + section_title만 인덱싱 → 본문 내용 검색 불가.
본문 검색을 가능하게 하는 실증 설계.

## 최종 결론

**scope 분리가 답이다.** title 검색과 content 검색을 하나의 엔진으로 합치면 가중치 합산에서 품질 저하.

```python
dartlab.search(query, scope="title")    # ngram bigram+trigram (현재 95% 유지)
dartlab.search(query, scope="content")  # word tokenizer + BM25 (신규)
```

## 핵심 발견

### 1. ngram은 본문 검색에 부적합 (002, 003)

| 토크나이저 | Stems | 크기 | 품질 (본문형) |
|-----------|-------|------|-------------|
| ngram (bigram+trigram) | 169,758 | 27MB | 노이즈 많음 |
| **word (공백+구두점)** | **48,715** | **3.9MB** | **명확** |
| kiwi 형태소 | 15,244 | 2.6MB | "구조조정" 실패 — 불안정 |
| substring | — | 0 | 정확하지만 랭킹 없음 |

**word 토크나이저가 최적**: 크기 7x 작음, 빌드 7x 빠름, cold start 0, 의미 단위 유지.

### 2. 필드 분리 BM25 — 합산은 실패 (005, 006)

가중치 `5×title + 2×section + 1×content` + 각 score 정규화 → **weak match 뻥튀기**.

예: "원재료 가격 급등" 쿼리에서 무관한 "개선기간 종료" 공시가 5.0점 상위.

### 3. scope 분리 — 성공 (007)

**scope="content" (BM25 단독) 결과:**

| 쿼리 | top-1 | 판정 |
|------|-------|------|
| 반도체 HBM 투자 | 한미반도체 사업보고서 (반도체 장비) | ✅ |
| 원재료 가격 급등 | 명인제약 사업보고서 — 원재료 매입 표 | ✅ |
| 환율 변동 리스크 | 오스테오닉 — "환율 변동 리스크 관리" 구절 | ✅ |
| 해외 매출 비중 | M83 — 수출/내수 매출 표 | ✅ |
| 신규 수주 공급 계약 | LS — 주요 계약 (EPC 수주) | ✅ |
| 경쟁 심화 마진 압박 | E1 LPG — 가격 변동 수익성 | ✅ |
| 구조조정 계획 | 스페코 — "구조조정 등을 통해" 구절 | ✅ |
| 공장 가동률 하락 | 원재료 표로 약간 drift | ⚠️ |

**8개 중 7개 완벽 매칭.** 제목만으로는 절대 찾을 수 없는 본문 내용 매칭 달성.

### 4. 증분 설계 (004)

**세그먼트 분할**: main (월 1회 리빌드) + delta (일 단위 증분)
- delta 빌드: 6,200문서 1초
- main 풀리빌드 (4M 문서): 약 18분
- 병합 검색: 0.4~2.6ms

## 스케일 추정

실측 (445K 문서, docs 100종목 + allFilings 30일):
- 빌드 시간: 2.1분
- 인덱스 크기: 545MB

4M 문서 선형 확장:
- 빌드: 약 18.7분 (월 1회 수용 가능)
- 크기: 약 4.8GB (HF 배포 가능)

## 제외된 방향

- **kiwi 형태소**: "구조조정" 등 분석 실패 — 불안정 → 제외
