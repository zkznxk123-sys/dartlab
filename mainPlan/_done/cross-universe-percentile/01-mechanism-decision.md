# 01 — 시각화 메커니즘 결정

> 운영자의 열린 질문: "크로스 백분위를 잘 볼 수 있는 매커니즘이 뭘까 — scan등급 상세보기 방식? 히트맵?"

## 결정

- **주 메커니즘** = **축별 행 × 유니버스 열 백분위 띠** (지표 1개 = 한 행, 행 안에 업종/시장/전체 N칸 띠를 나란히).
- **드릴다운** = **행 클릭 → 그 지표의 `DistCurve` 분포곡선**(band 있는 유니버스 = 업종; 시장/전체는 Phase 2 band 도착 시 자동).
- 히트맵·레이더 스택·소형다중 **기각**.

## 후보 비교 (적대적)

### Strawman "큰 히트맵 하나 던지면 된다" — 기각
1. **터미널 패키지에 히트맵 컴포넌트가 없다.** `HeatmapChart`/`ChartRenderer`/`peer-matrix` 는 `ui/web`(레거시) 잔재이고 `ui/packages/surfaces`(터미널 본진)에 **없다**. "손수 차트 금지" 규칙상 히트맵은 *신규 손수 컴포넌트* = 규칙 위반. 실재 재사용 자산은 `DistCurve`·`RadarChart`·모달 셸·RightStack 띠 마크업뿐.
2. **셀당 분포가 안 보인다.** 백분위 65 와 67 이 같은 색 → 의사결정에 무용.
3. **정성셀 색 의미 붕괴.** 거버넌스 A~E 를 0~100 백분위 색 스케일에 칠하면 가짜 정량화(02 KILL #3).
4. **허위정밀.** 12지표 × 3유니버스 = 36셀을 정밀 색으로 칠하면 scan-grade 가 reject 한 "12등급 허위정밀" 의 격자판. 대부분 셀이 분기 latency·동일가중↔시총가중 혼재.

### 레이더 스택(유니버스마다 레이더 겹치기) — 기각
12축 극좌표 + 3유니버스 오버레이 = 스파게티. 게다가 scan-grade-explainer 가 **이미 레이더를 종합요약으로 사용** → 역할 중복.

### 채택안이 충족하는 것
- **스캔성("한방에")**: 각 행이 업종/시장/전체 칸을 나란히 → "이 회사는 업종선 상위, 전체선 중위" 가 가로로 즉시 읽힘. 히트맵의 "한방에" 를 *행 단위로 분해*해 보존하되 분포는 안 잃음.
- **드릴다운**: 행 클릭 → `DistCurve {band,value,p,unit}` 에 band 만 유니버스별로 넘기면 끝. 신규 컴포넌트 0.
- **정성+정량 혼재 명시**: 정량 행 = 백분위 띠. 정성 행 = 띠 대신 **등급 칩 + 동급 비중 막대**, 색은 백분위색이 아니라 등급 톤(tUp/tDn)으로 분리 → "백분위인 척" 차단. 데이터 없는 유니버스 칸 = "—".
- **좁은 폭 폴백**: N열이 안 들어가면 유니버스를 상단 **세그 토글 1개**로 접고 단일열 띠로 폴백 — 행 구조 불변.
- **무손상 확장**: 1열(업종, Phase 1) → 3열(시장·전체 추가)이 레이아웃 손상 없이 자동 채움. 데이터 도착 시 칸만 채워짐 = 히트맵 대비 결정적 이점.

## 가격(밸류에이션) 격리
PER/PBR/PSR 백분위는 펀더멘털 격자에 섞지 않는다(02 KILL #2 — 가격을 펀더로 위장 차단). 필요 시 다이얼로그 하단 **별 소격자 + "가격 지표(시장 시세 기반)" 캡션**으로만, 펀더 행과 물리적으로 분리.
