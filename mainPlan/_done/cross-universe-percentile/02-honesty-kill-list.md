1. **36셀 색 히트맵** — 허위정밀(scan-grade "12등급" reject 와 같은 병). 대안 = 축별 행 띠(01).
2. **밸류에이션(PER/PBR/PSR) 백분위를 펀더 격자에 합류** — 가격신호를 펀더로 위장(scan-grade composite 0.32 가격오염과 동형). 가격은 **별 소격자 + 캡션**으로만 격리(01 가격 격리).
3. **범주형 등급(A~E·자본환원·부채위험)을 0~100 색칠** — 가짜 정량화. 등급 칩 + 동급 비중만(00 정성 처리).
4. **종합 verdict / composite / 매수매도 / "종합 1위" / "좋은 주식"** — 그건 `scan-grade-explainer` 의 *종합평가 레인*. 본 화면은 **분포 사실(상위 N%)만, 판정 0**(운영자 2026-06-14 레인 분리).
5. **소속지수 백분위 위조** — 구성종목 멤버십 데이터 부재(00 ④). "시총 상위 N개 = KOSPI200 근사" 같은 우회는 부정확한 위조 = 정공법 위반. BLOCKED 는 BLOCKED 으로 명시한다.

## 필수 가드 (KILL 후 잔존분)

- **분포 출처 라벨 강제**: "industryStats = KSIC섹터 · 동일가중 · 상장 primary ≠ KRX 시총가중 업종지수". 셀/유니버스마다 출처 + as-of.
- **cross-sector caveat**: 전체상장사 유니버스는 금융·제조 혼재라 ROE/마진 백분위가 섹터 의존. "전체상장사 ROE 상위 10%" 는 *맥락이지 헤드라인 아님* — **업종을 앵커(기본)로, 시장/전체는 넓은 잣대로** 명시. (이 뒤집힘 자체가 화면의 통찰이다.)
- **n<10 분포 숨김 + "n=N" 노출**: 표본 부족 유니버스/지표는 곡선 숨기고 표본수 표기.
- **mean±std 박스 금지, percentile band(p10~p90)만**: outlier heavy(ROE·opMargin) 검증됨.
- **결손 = 축 제외(0대체 금지)**: `.filter(v 존재)`. 0 채우면 "취약" 오독.
- **provenance/as-of 칩 없는 셀 = 회귀.**
- **자동 push 금지**: UI 변경이라 운영자 눈검수 + 명시 승인 후에만(feedback_ui_rules).

## 살아남는 정확한 경계선

이 기능이 사는 형태 = **scan-grade(종합평가·판정)와 분리된, 유니버스 교차 *분포 사실* 화면.** "4유니버스 60셀 색 히트맵 + 종합점수" 로 부풀리면 KILL 들에 걸려 전체 KILL. "단일 산식(pctRank) × 모집단 3종 + 분포곡선 + 한계 라벨" 로 *축소*하면 산다. 차별은 더하기가 아니라 빼기에서 — *더 적은 잣대로 더 자주 뒤집힘을 드러내는 것*.
