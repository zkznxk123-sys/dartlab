# 04 — 완성 범위 · 경계 연결 · 성공지표

> 정공법 기본 = 완성형. 단계로 얇게 쪼개지 않는다. 지금 데이터로 만들 수 있는 건 **한 빌드에 전부**. 단 하나의 제외는 *데이터가 물리적으로 없어서* 막힌 것뿐(소속지수).

## 완성 범위 (한 빌드)

| 구성 | 내용 | 데이터 |
|---|---|---|
| 다이얼로그 셸 | 우측 "업종 내 백분위" 헤더 → 상세 보기 → 모달 | 셸 라이브(재사용) |
| 3 유니버스 백분위 | 업종 / 소속시장(KOSPI·KOSDAQ) / 전체상장사 — 행 × 열 띠 | `EcoNode` 전종목 raw 재필터 · **새 데이터 0** |
| 분포곡선(DistCurve) | **전 유니버스** — 업종=industryStats(기존), 시장·전체=**노드 배열에서 라이브 5분위 계산** | 라이브 · **prebuild 불필요** |
| 정량 12지표 | 수익성·성장·안정성·유동성·효율·이익질 | engine.ts 현행 |
| 정성 지표 | 거버넌스 등급 등 — 등급 칩 + **동급 비중**(이 유니버스 중 B등급 28%) | 노드 배열 카운트 · 라이브 |
| 가격 격리 | PER/PBR 별 소격자 + 캡션(펀더 격자 합류 금지) | 02 KILL #2 |
| 한계 라벨 | 분포출처·cross-sector caveat·n·provenance/as-of | 02 가드 |

핵심: **시장/전체 분포곡선도 prebuild 없이 라이브다.** DistCurve 는 `{p10,p25,median,p75,p90}` 만 필요한데, 백분위 계산에 쓰는 *바로 그 모집단 배열*을 정렬해 5분위를 뽑으면 된다(정렬 ~2664값 × 지표 × 유니버스 = 다이얼로그 열 때 1회, 무시 가능). 업종은 기존 `industryStats` 밴드 유지(scan-grade 다이얼로그 미회귀), 시장·전체는 같은 모집단에서 라이브 산출 → 백분위와 자기일관.

## 단일 제외 — 소속지수 (데이터 게이트, 단계 아님)

소속지수(KOSPI200·코스닥150 등)는 완성형 *밖*이다. 단계로 미룬 게 아니라 **구성종목 멤버십 데이터가 물리적으로 없어서**(00 ④: KRX 지수=OHLCV만) 지금 만들 수 없다. "시총 상위 N개 근사" 우회는 위조라 금지(02 KILL #5). 다이얼로그엔 소속지수로의 **link-only 자리표시**만 두고, 별도 *구성종목 수집 졸업게이트* 통과 시 같은 격자에 열이 자동 추가된다(레이아웃 무손상). 데이터 확보 전까지는 빈 칸으로 둔다.

## 인접 PRD 침범 KILL
컨센서스·목표주가·"적정주가"·멀티심볼 보드·시계열 백분위 추이(2020+)·새 차트엔진·추천/매수신호 — 전부 인접 PRD 소유 또는 설계 위반. 본 화면 = *1사 · 단일시점 · 분포 사실*.

## 성공지표 (셀·유니버스 개수 아님)
1. **유니버스를 바꾸니 순위가 뒤집히는 회사**를 사용자가 발견하는 빈도 — 이 화면 고유 통찰.
2. 다이얼로그 → compare / 워치 / 산업점프 **연결 클릭률**(허브로 기능하는가).
3. 결손 축이 0으로 위장되지 않고 빠지는 비율 **100%**(결손 표기 회귀 0).

"더 많은 유니버스" 가 아니라 **"더 적은 잣대로 더 자주 뒤집힘을 드러냄"** 이 성공.
