# 캐러셀 100개 후보 백로그

작성일: 2026-06-19

## 운영 전제

이 문서는 발행 확정표가 아니라 후보 풀이다. 모든 행의 기본 상태는 `live-check 필요`다. 제작 직전에는 공식 IR, DART/EDGAR, 거래소 공시, 회사 보도자료, 신뢰 가능한 보도 중 최소 1개 이상으로 숫자와 사건을 다시 확인한다.

릴스는 보류한다. 산출물은 1080x1350 캐러셀만 만든다.

공개 캐러셀 표기는 회사명, 주제명, 종목코드 6자리로 끝낸다. `D01`, `d01`, `E27`, `X03` 같은 내부 분류번호는 발행 순서와 무관하므로 폴더명과 QA 기록에서만 관리한다.

## 선정 루프

1. 뉴스 에디터: 최근 화제성, 사람들이 궁금해할 질문, 저장할 이유를 제안한다.
2. 팩트체커: 과장 프레이밍, 투자판단처럼 보이는 결론, 1차 근거 필요 항목을 차단한다.
3. 비주얼 디렉터: 4:5 화면에서 큰 텍스트와 이미지가 살 수 있는지를 본다.
4. 편집자: `강한 첫 장 훅 → 실제 사실 → 왜 중요 → 재미있는 숫자 → 명확한 마무리` 로 6~12장을 구성한다. 첫 장은 주제별로 새로 기획하고, 조회가 잘 나온 과거 문법을 그대로 복제하지 않는다.
5. 훅 라이터: 첫 장이 `audienceQuestion`, `coreAnswer`, `tension`, `soWhat` 중 최소 두 축을 담는지 본다. 요약형 커버는 실패다.
6. 렌더 후 이미지 검사: 모든 카드를 반드시 한 장씩 직접 열어 본다.

## 상태값

- `live-check 필요`: 최신 수치와 사건 확인 전.
- `ready-brief`: 1차 근거와 훅이 맞고 이미지 모티프가 잡힘.
- `in-production`: hook.json, caption, source_report 작성 중.
- `rendered`: PNG 렌더 완료.
- `needs-context-gate`: 과거 제작·검수 표기가 있으나 현재 전체 맥락 게이트 통과가 확인되지 않음.
- `passed-visual`: 현재 `checkCarouselEditorial.py` + 전체 배치 감사 + 이미지 직접 검수 완료.
