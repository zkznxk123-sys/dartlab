# 00. 등급 수 판정 (OQ4) — "12밴드" 허위정밀 reject

상태: PRD v0.1 (2026-06-14, 코드 실측 + 적대검증 #1 survives=true + 4렌즈 만장일치 falsePrecisionRisk:high)
범위: 운영자 "스캔등급 12단계 확장 가능한가" 질문에 대한 판정 + 대안.

---

## 1. 전제 정정 — "12단계 스캔등급"은 코드에 부재 (strawman)

운영자 직감과 달리, 단일 12단 스캔등급은 *실재하지 않는다*. 실측 3층 구조(`ui/packages/surfaces/src/terminal/lib/engine.ts`):

| 층 | 위치 | 구조 |
|---|---|---|
| ① 축별 등급 | `engine.ts:44-53` `GRADE_SCALE` | 7축 각자 *독립* categorical 스케일 — prof/qual/liq 5단·growth 5단·gov A~E 5단·**audit 3단**·stab 6단·debt 4단 |
| ② 종합 등급 | `engine.ts:268-300` `verdictOf` | 정확히 **5밴드** — STRONG(≥74)/SOLID(≥60)/NEUTRAL(≥46)/CAUTION(≥32)/WEAK |
| ③ 신용 등급(별개) | `deriveCredit` (dCR) | **14밴드** AAA~CCC — 스캔등급 아님 |

즉 "12로 확장"의 대상이 모호하다. 3해석 모두 판정한다.

## 2. 12밴드 3해석 판정표 — 전부 REJECT

| 해석 | 내용 | 판정 | 근거 |
|---|---|---|---|
| **(A) 7→12 축 증식** | 표시 축 개수를 7→12로 | **REJECT** | `EcoNode` grade 필드 7종이 데이터 천장. 새 축은 `src/dartlab/scan` 엔진 신설 + ecosystem JSON 재빌드 필요(브라우저만으론 불가) + anti-clutter 위반. `cfPattern` 1종만 미등급 필드로 잔존(증식 아님) |
| **(B) 축별 5→12단 세분** | 각 축 등급을 더 잘게 | **REJECT(허위정밀)** | 입력이 단일 파생값(예: `_gradeProfitability=max(opMargin,roe)`)→5 hard threshold 계단함수(`profitability.py:61-90`). 12단 쪼개면 의미 없는 경계, 공시 데이터 미지지, 결손 0대체 유발 |
| **(C) 종합 composite 12밴딩** | composite 0~100을 12밴드로 | **REJECT(허위정밀)** | §3 |

## 3. composite 12밴딩이 허위정밀인 이유 (해석 C 상세)

`composite`(`engine.ts:278`) = `0.44·gradeAvg + 0.18·valScore + 0.14·momScore + 0.18·twScore + 0.06·risk`.

**(a) 외관 연속성은 계단함수 평균 인공물**: `gradeAvg`는 5~6단 categorical 등급의 평균, `audit`는 `{1.0, 0.5, 0.0}` 3값뿐, `valScore`/`momScore`는 4버킷. 0~100으로 매끈해 *보이지만* 진짜 해상도 천장은 **5~7밴드**다. 12밴드로 쪼개면 인공적 평균의 소수점을 등급 경계로 위장하는 것.

**(b) ★결정적 2차 위반 — 가격 오염**: composite는 `momScore`(1Y 수익률) + `twScore`(섹터 순풍) = **0.32 가중의 가격 신호**를 포함한다. 12밴드 "펀더멘털 등급"으로 발간하면 *가격+펀더 혼합을 펀더 등급인 척 위장*한다 — dartlab 설계 원칙(등급=판정, 가격≠펀더) 정면 위반.

적대검증 #1 survives=true. 4렌즈 만장일치 `falsePrecisionRisk: high`.

## 4. 권고 — 신설 스케일 0으로 "12 갈증" 충족

운영자의 "더 세밀한 등급" 욕구는 *허위 등급 신설* 없이 충족한다:

1. **종합 = 5밴드 유지**(STRONG~WEAK). 색 밴드로.
2. **연속 composite 0~100 숫자를 그대로 노출**(밴드 옆에). 5색 밴드 + 숫자 = 허위정밀 회피 *동시* 정보량 보존. 사용자가 "SOLID 62 vs SOLID 59"를 직접 본다(가짜 12등급 경계 없이).
3. **이미 있는 dCR-14 신용등급을 다이얼로그에 *합류 표시***(신설 스케일 0). 진짜 14단 해상도가 필요한 사용자는 신용등급에서 얻는다.
4. **결손 축 `.filter(v존재)` 제거 유지**(0대체 금지 = 설계대로).

> 핵심: 등급 *수*를 늘리는 게 아니라, *이미 있는 연속 점수 + 별개 14밴드 신용등급*을 있는 그대로 드러내 해상도 욕구를 충족한다. 새 등급 산식·새 밴드 신설 0.

## 5. 운영자 보고 문구 (정정 동반)

"12단계로 확장"은 *기능적으론* 가능하나(버킷 경계만 늘림) **허위정밀이라 권하지 않는다**. ① 그런 단일 12단 등급이 현재 부재하고 ② composite의 진짜 해상도는 5~7밴드이며 ③ composite엔 가격 신호 0.32가 섞여 12밴드 "펀더 등급"은 가격을 펀더로 위장한다. 더 나은 대안 = **5밴드 + 연속 숫자 노출 + 신용 dCR-14 합류**(신설 0, 정보량 더 많음). → 01 다이얼로그가 이 노출을 담는다.
