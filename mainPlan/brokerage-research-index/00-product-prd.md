# 00 — 제품 PRD (정본)

## 1. 무게중심 (한 문장)

세상에 흩어진 증권사 리서치 리포트를 — 본문을 *생산하거나 호스팅하지 않고* — 각 증권사 **공개 게시판에서 메타데이터(제목·URL·발간일·증권사·종목)만 자체 스크래핑**해 한데 모아, **날짜순 전체 / 종목별 / 검색** 세 패턴으로 호출 가능한 **합법 링크 인덱스**를 gather 에 짓는다. 본문은 우리 화면에 담지 않고 *원본으로 링크아웃*한다.

## 2. 비전

> 네이버 리서치는 33개 증권사와 **제휴 바터**(종목토론방 API ↔ 리포트)로 *본문까지* 무료로 보여준다. 그러나 그 리포트가 **맞았는지는 아무도 채점하지 않는다**. dartlab 은 링크 인덱스 위에 **1차 데이터(공시 실적·주가·금투협 괴리율)로 애널리스트·증권사를 채점하는 검증 레이어**를 얹는다 — 네이버·FnGuide·한경 누구도 구조적으로 안 하는 칸. 우리는 리포트를 *모아 보여주는 자*가 아니라, 리포트를 **평가하는 자**다.

이 PRD 가 소유하는 영토 = **리서치 리포트의 메타 인덱스** (+ phase 3 채점). 본문 콘텐츠 생산·호스팅은 **영원히 비(非)영토**. 증권사 거래 API 통합(BYO 토큰 자동투자)은 *별도 제3엔진* — `02 §경계` 포인터만.

## 3. 문제 / 왜 지금

- 리서치를 한데 모아 보여주는 소비자 무료 표면 = 사실상 **네이버 독점**(+한경, 단 한경은 로그인벽으로 후퇴 — 본 세션 실측). 둘 다 **제휴 계약망 게임**이라 신규가 같은 방식(본문 재배포)으론 못 이긴다.
- 회사별 목표주가 컨센서스는 **FnGuide(에프앤가이드) 유료 독점**(Data Feed FTP/API). 무료 1차 직수집 경로 없음.
- 우리가 합법으로 *직접* 가져올 수 있는 건 단 둘 — (a) 각 증권사 **공개 게시판의 리포트 메타**(제목·링크·날짜), (b) **금투협 전자공시**(dis.kofia.or.kr)의 증권사단위 괴리율·투자의견 비율. 둘 다 1차·규제/공개.
- 진짜 빈칸: 아무도 **"이 애널/증권사가 과거에 맞았나"를 채점**하지 않는다. dartlab 은 1차 데이터(공시 실적·주가)를 이미 쥐고 있어 이걸 할 수 있는 **유일한 위치**.

## 4. 법적 기반 (이 제품의 토대 — 본 세션 출처검증)

1. **링크는 합법** — 대법원 2008다77405·2010(2009다4343): 링크는 '전송'이 아니라 복제·전송권 침해 아님(한국 확립 판례). → 제목+URL 링크 인덱스는 **라이선스 불필요**.
2. **본문 호스팅만 라이선스** — 우리 화면/서버에 PDF·전문을 담으면 복제·전송권. 네이버가 본문을 보여줄 수 있는 건 *줄 카드(토론방 API)*가 있어서다. 우린 그 카드가 없다. → **절대 본문 안 담는다.** 링크아웃만.
3. **목록 통째 스크랩 = DB권 회색**(저작권법 제93조; 크롤링 판례가 민사[부정경쟁 인정]·형사[DB권 침해 아님]로 갈림 — 사안별). → 네이버/한경 목록 통째 긁기 **금지**, **각 증권사 공개 게시판 자체 인덱싱**만.
4. **snippet 최소·사실 또는 우리 생성** — 증권사 요약을 길게 verbatim 저장 = 복제 회색.
5. **robots.txt·로그인벽 존중** — 차단·로그인 요구 사이트 크롤 금지(미래에셋 "무단복제·배포 금지" 명시 등).

## 5. 차별화 (왜 다른 데선 못 얻나)

- **단순 aggregator = 네이버 하위호환**(본문 못 보여줌 + 제휴 없음 → 구조적으로 못 이김). 그래서 *그 자체는 목표가 아니다*.
- **우리 칸 = 링크 인덱스 + 채점**: ① 애널 목표주가 **과거 적중률** ② 증권사 **괴리율·매수편향**(금투협 연료) ③ 컨센서스 vs **실제 공시실적 갭**. 한경·FnGuide·네이버 누구도 안 한다.
- **gather SSOT 정합**: 수집=gather 단일 작업대, sync(online)→HF→prebuild(offline) **기존 레일 그대로**. 새 인프라 0, 새 외부 의존 0.

## 6. NEVER-CLAIM (위반 시 제품 폐기/법적 리스크)

1. 본문(PDF·전문) **호스팅·임베드·재배포 금지** — 제목·URL·최소 snippet 만.
2. 네이버/한경/FnGuide **목록 통째 스크랩 금지** — 각 증권사 공개 게시판 자체 인덱싱만.
3. **robots.txt 차단·로그인벽 사이트 크롤 금지**.
4. 리포트 내용을 **"매수/매도 신호·판정"으로 재가공 금지** — 우리는 메타 + 채점(사실 수치)만, 추천 톤 0.
5. 회사별 목표주가 컨센서스를 **무료로 가진 척 금지**(FnGuide 유료) — 직수집 가능한 건 금투협 증권사단위 통계뿐.
6. **gather 직행 금지** — `tests/_attempts/brokerageIndex/` 졸업게이트 경유(개념확립·ticker 커버리지 실측 후에만 본진).
7. **별도빌드 금지** — `.github/scripts/sync` 는 gather 함수 **호출만**, 스크랩·파싱 로직 재구현 금지(CLAUDE.md ⛔ 공동작업대 SSOT).
8. 외부 콘텐츠 **untrusted wrap 누락 금지**(snippet → `wrapExternal()`).
9. **ticker 0-fill·추측 금지** — 제목 해소 실패 시 `null`(산업·시황 리포트는 애초에 null first-class).

## 7. 성공 기준

- _attempts: 5~6개사(미래·한투·NH·삼성·KB·키움) **안정 스크랩** + ticker 해소 **커버리지 실측 보고**(README 박제).
- `g.brokerageReports()` 가 **날짜순 · 종목별 · 검색** 3패턴 × **hf/live** 2모드 동작.

- HF `brokerageReports`(public) 백필 + prebuild 직독 동작, sync 가 gather 호출만(별도빌드 0).
