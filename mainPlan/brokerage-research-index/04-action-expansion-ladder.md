# 04 — 액션 확대 사다리 (Action Expansion)

> "데이터 인덱스"에서 멈추지 않고, 인덱스가 **운영 → 사용자 → AI 액션**을 구동하는 데까지 단계적으로 확대한다. 각 L 은 **앞 L 위에서만 선다**(순서 불가침 — 인덱스 없이 채점 없고, 채점 없이 에이전트 액션 없다).

## 사다리 한눈

| L | 이름 | 무엇을 하나 | Phase(02) |
|---|---|---|---|
| **L0** | 데이터 | 인덱스 queryable (날짜순·종목별·검색) | P0~P1 |
| **L1** | 자동 갱신 | cron 증분 → HF, 사람 개입 0 | P2 |
| **L2** | 자동 운영 헬스 | 깨짐 감지·자동 강등·freshness·운영자 요약 | P2 (03) |
| **L3** | 사용자 향 액션 | 구독 알림·터미널 레일·MCP tool | P2.5 |
| **L4** | 채점 액션 | 적중률 자동 채점·메타 라벨 | P3 |
| **L5** | 에이전트/거래 경계 | AI 종합·거래 신호 | *이 PRD 밖* |

## L0 — 데이터 (base · 00~02)

`g.brokerageReports()` 인덱스가 세 패턴으로 응답. 수동 호출. *이 사다리의 바닥.*

## L1 — 자동 갱신 (03 §1~3)

cron stage → 증분 → HF push. 사람 개입 없이 매일 최신 유지. "데이터가 살아 있다."

## L2 — 자동 운영 헬스 (03 §4~6)

셀렉터 깨짐 감지 · 증권사 자동 강등 · freshness 노출 · 운영자 요약. **"방치해도 안 썩는다."** ← 스크래핑 제품이 죽는 가장 흔한 이유(소스 변경 방치)를 시스템이 먼저 잡음.

## L3 — 사용자 향 액션 (인덱스가 행동을 구동)

인덱스가 단순 조회를 넘어 *사건*을 만든다:

- **종목 구독 알림** — 사용자 watchlist 종목에 새 리포트 뜨면 digest. (인덱스의 신규 `report_id` delta 가 트리거)
- **터미널 "관련 리서치" 레일** — 회사 조회 시 `ticker` 필터 인덱스를 자동 노출(원본 **링크아웃**, 본문 호스팅 0). UI push = **운영자 승인 게이트**(CLAUDE.md ⛔).
- **MCP tool 노출** — `ai/tools/` 에 brokerageReports query tool → AI agent 가 "이 종목 최근 리서치"를 질의·인용(외부 본문 **untrusted wrap**). 새 능력은 `ai/tools/` 에만(graph 회귀 금지 — [[feedback_no_graph_regression]]).

## L4 — 채점 액션 (P3 · 차별화 본체)

인덱스 위에서 **우리만 할 수 있는 것**:

- 새 리포트 → 목표주가·투자의견 추출 → 누적.
- **금투협 괴리율 + 실제 공시 실적**과 대조 → **애널리스트·증권사 적중률 자동 채점**.
- "이 리포트 발행사는 과거 N개월 목표주가 X% 괴리 / 적중률 Y%" **메타 라벨 자동 부착**.
- 별도 게이트(P3) — 인덱스(L0~L2) 없이는 채점 불가 → 순서 고정.

## L5 — 에이전트 / 거래 경계 (이 PRD 밖)

- AI 자동 리서치 브리핑(채점 포함 종합).
- **증권사 거래 API 통합 제3엔진**(BYO 토큰)과 신호 연결 — 별도 엔진. *자동 실행*은 투자일임/유사투자자문 규제 경계 → **사용자 확인 단계 필수**.
- 이 PRD 는 L5 를 *구동하지 않는다*(경계 포인터만).

## 액션별 불변 가드

- 모든 사용자/AI 노출 = **메타 + 링크 + 채점(사실)만**. "매수/매도 신호" 재가공 금지(00 NEVER-CLAIM 4).
- 외부 본문 인용 = **untrusted wrap**.
- UI 노출 = **운영자 push 승인**.
- 사다리 위로 갈수록 강해지지만, **L 건너뛰기 금지** — L2 헬스 없이 L3 알림 깔면 깨진 데이터로 알림 보내는 사고.
