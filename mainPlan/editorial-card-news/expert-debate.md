# 에디토리얼 카드뉴스 — 전문에이전트 토론 기록

> 파이프라인 SSOT = [`README.md`](README.md). 본 문서는 그 설계가 *어떻게 나왔는지* — 2라운드 멀티에이전트 토론 + 적대검증의 기록.
> 방식: 다관점 병렬 생성 → 적대적 검증가가 무비판 수용 거부하고 *깎음* → 운영자 실측 판정 반영.

---

## Round 1 — 파이프라인 4관점 토론 (2026-06-18)

**질문**: 터미널+인스타 에디토리얼 카드뉴스를 자동으로 채울 "핵심 기업 스토리"를 어떻게 뽑고/이야기로 만드나.

### 4관점 핵심 주장

1. **에디토리얼 전략가** — "카드는 발명이 아니라 증류". blog 리포트(`blog/05-company-reports/*`)가 이미 역설 헤드라인을 써뒀다(고려아연 "50년 만에 첫 적자인데 영업이익 사상최대"). 2-소스 하이브리드: blog=1차 우물(거의 자동), 엔진 신호=미커버 후보 발굴. **숫자 역설은 자동, 인과 서사는 사람 confirm.**
2. **엔진 아키텍트** — `story/narrative.py::detectThreads`(역설 7패턴, NarrativeThread{title=인과, evidence[]=payoff, severity=톤})가 이미 "한 줄 훅"의 90%를 결정론 생성. `narrate.py`가 값→한국어 레이블 변환 SSOT. 신설은 얇은 어댑터만.
3. **파이프라인 엔지니어** — 단일 SSOT `cards.json` → 인스타 PNG(기존 Remotion `TopPhotoLayout` 재사용) + 터미널 Svelte 라이브 렌더. landing `routeLoad.ts`+runtime `dartlabData.ts`에 한 줄씩 = HF-first 캐시 공짜 상속.
4. **비주얼 디자이너** — 기존 `TopPhotoLayout`이 레퍼런스와 80% 일치. 신규 `editorial` 레이아웃 분기 + `AccentText accentColor` 주입 + CSS grayscale. 신규 컴포넌트 최소.

### 적대검증 (PM 겸 적대가)의 컷 — Round 1을 깎다

> 4관점 합의를 무비판 수용하지 않고, 데이터를 직접 열어 치명적 오류를 찾았다.

- **🔴 최대 발견 — sub 자동추출은 거짓**: `ai.verdict` 길이가 **이중분포**(초기 30편 100~160자 vs 최근 100+편 878~1844자, LG화학 600자 통문장). "verdict→payoff 자동" 가정이 데이터 절반에서 깨진다. **가장 품질 높은 최근 리포트일수록 카드가 더 심하게 깨진다.** → title만 자동, sub는 사람 30분.
- **과설계 적발**: 엔진 어댑터 3종(cardSlots/queue/게이트)은 `_attempts` 졸업 전 본진 금지 위반 + 5장 목표엔 전종목 큐 불필요. → MVP에서 제거, blog→cards.json 스크립트 1개로 충분.
- **비주얼 과대추정**: `TopPhotoLayout`이 이미 대부분 구현. 신설 = grayscale 한 줄 + 오렌지 날짜 + accentColor. 신규 파일 4개 → **2개**.
- **인스타 먼저**: 터미널 CardFeed는 신규 도달 0(자산 재활용) + UI push 승인게이트. 인스타가 가치 증명.
- **engineRef 게이트 완화**: blog는 이미 사람 검증 출판물 → `sourceReport` 링크로 충분, 렌더차단 게이트는 engine-first까지 보류.

**결론**: blog-first · 인스타 먼저 · 비주얼 최소패치 · 자동화는 "수동 1편→검증→자동화".

## Round 2 — "확실한 후킹" 3관점 토론 (2026-06-18)

**촉발**: 운영자가 첫 커버 "5번 죽을 뻔한 회사가 모회사 삼성전자를 앞질렀다"를 **"별로 와닿지 않는다"** 반려. "한 분기에 19조"로 교체하니 강해짐. → **훅 품질이 make-or-break. 운빨이 아니라 보장으로.**

### 3관점 핵심 주장

1. **후킹 카피/심리 전문가** — 호기심 갭(Loewenstein) 이론으로 분해: **추상 서술은 갭을 흐리고, 구체 숫자는 갭을 날카롭게 한다.** "앞질렀다"는 해석 부담을 독자에 떠넘겨 죽고, "19조"는 자기완결. 후크 taxonomy 8종(숫자충격·역설·버려진것의반전·꼴등역전·미스터리질문·의외정체·권력불의·정체성위협). dartlab 우위 = "셀수록 설득력 있는 유일한 구조"(데이터가 떠받침).
2. **데이터→후크 엔진 아키텍트** — 회사 1개 → 강후보 ≥5개를 4개 기존 자산(frontmatter ai.* + detectThreads + narrate._classify + scan RankInfo)에서 결정론 생성. **fallback 사다리**(숫자충격→순위→변화율→정체반전→미스터리질문)로 밋밋한 대형주도 보장, 5단계 전부 비면 스킵. 신설 = `tests/_attempts/hookEngine/` 졸업 후 `story/hooks.py` 1모듈.
3. **적대적 후크 심사관** — **scoring 루브릭 7축(0~100)** + **reject-gate**(무숫자·과장어·갭자폭 hard block) + **generate-many-score-pick**(≥75 자동/60~74 보강/＜60 스킵). 운영자 두 사례를 채점해 **42점 vs 92점**으로 직관을 정량 재현(분리력은 숫자충격 A축 6→22, 호기심갭 B축 9→18).

### 적대 심사관의 8장 실측 재채점 (운영자 직관의 정량화)

| 카드 | 점수 | 판정 | 보강 |
|---|---|---|---|
| ① 커버 "망하기 직전…19조" | **92** | S 유지 | 분할 시 95 |
| ② "다섯 번 죽을 뻔" | **58** | ⚠ 차단경계 | "다섯 번 죽을 뻔하고 이름은 세 번 바뀌었다"(두 구체 사실) |
| ③ "비결은 HBM 한 단어" | 71 | 보강 | "19조의 비결은 단 **세 글자**, HBM"(앞 카드 연결+갭증폭) → 80 |
| ④ "58.4%" | 64 | 보강 | "100원 팔면 58원이 남는다는 뜻"(의미번역) → 79 |
| ⑤ "빚보다 현금이 많다" | 73 | 보강 | "빚 다 갚고도 **7.4조**가 남는다"(숫자화) → 80 |
| ⑥ "NVIDIA 한 곳이 매출 27%" | **81** | S 유지 | "매출 4분의 1이 단 한 곳, NVIDIA에서" → 84 |

→ 보강 카피 전부 적용 완료(`E01-000660-skhynix-editorial/hook.json`). 전부 frontmatter 검증 숫자.

### 적대 심사관의 자기비판 (스코어링의 한계)

- **측정 가능한 약한 훅 생존, 측정 불가한 천재 훅 사망** → 60~74 "보강 큐"를 *천재 훅 구제* override 슬롯으로. override 실측 성과 = 루브릭 진화 입력.
- **Goodhart 함정**(루브릭 알면 "조 단위+12자+강조1개"만 양산→피드 둔감화) → type 다양성 쿼터(회차당 같은 type ≤ 60%).
- **정확성 게이트 과차단**("역대 최대"가 사실인데 G4 오탐) → 검증된 사실 기반은 출처 첨부 시 화이트리스트.
- **불변선**: 강함은 양보 가능, **정확성(G4 과장어·G7 미검증)은 override로도 못 뚫는다.**

## 합의된 불변 원칙 (양 라운드 공통)

1. **숫자·사실 = 100% 결정론(엔진 ref), LLM = 문장 윤문·강조단어 선택만.** 새 수치 생성 금지(string match 검증).
2. **약한 훅은 발행하지 않는다** — 약한 발행보다 스킵이 낫다.
3. **"이 훅의 충격이 사라져도 숫자는 사실인가?"** Yes만 발행. 충격을 위해 맥락을 빼면 클릭베이트.
4. **자동화 경계**: 후보생성·스코어링·숫자 자동 / 키포인트 큐레이션·인과 프레이밍 사람.
5. **`_attempts` 졸업 게이트**: Hook Engine 본진(`story/hooks.py`) 직행 금지, 개념확립·데모 후.
