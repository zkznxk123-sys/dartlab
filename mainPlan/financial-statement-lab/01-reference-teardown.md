# 01. 레퍼런스 분해 — iTooza V차트 + 버틀러 Butler

상태: 비전 PRD v0.1 (2026-06-13)
목적: 두 레퍼런스를 *복제 대상*이 아니라 **검증된 축 분류·series 조합·UX 패턴 사전**으로 분해한다. 무엇을 가져오고(TAKE), 무엇을 버리고(REJECT), 무엇을 미루나(DEFER). iTooza 50차트 전수 census 정본 = 메모리 `reference_itooza_vchart_system`.

## 1. iTooza V차트 — 구조

가치투자 서적("재무제표 해결사 V차트" 정연빈 · "가치투자가 쉬워지는 V차트" 최준철) 계보의 재무차트 도구. 전수 HTML census 기준 **50 chartId**.

- **6 중분류**: 수익성 / 안전성 / 자산구조와배당 / 효율성 / 현금흐름 / 밸류에이션.
- **3 기간모드**: `dcy`(연환산/TTM) · `dy`(연간) · `db`(분기).
- **업종별 차트셋**: 제조/비금융 37(수주잔고 노출시 38) · 은행 26 · 보험 24 · 증권 26. 은행은 예대마진·자기자본비율·대손비율·ROA로, 보험은 보험료수익·자산운용수익으로, 증권은 부문별손익·신용/주식담보대출로 갈아끼운다.
- **핵심 철학**: "한 화면에 복수 series를 겹쳐 원인/결과를 보게 한다" — 금액+비율, 주가+기초체력, 회전일수+수익성 같은 *해석 쌍*. 데이터 응답은 Chart.js config(`labels`=YYYYMM x축, `datasets[]`=series, `yAxisID`=좌/우축).

### 우리가 이미 가진 것 (놀랍게도 많음)
MiniFinChart는 iTooza의 핵심 메커니즘 — *금액 막대 + 비율 선, 이중축 overlay* — 을 이미 구현했다. 우리 ~40카드는 iTooza 6분류를 5탭으로 압축하고, iTooza가 *없는* 것까지 가졌다: 만기 사다리·감사보수 독립성·자사주 수량 흐름·보수배율·OCI bridge·인력/이사회. **이건 클론 타겟이 아니라 강점이 직교하는 두 레퍼런스의 갭 감사다.**

### iTooza 시그니처 중 우리가 *없는* 것 — 고가치 vs 덕지덕지

| 순위 | iTooza 차트 | 가치 판정 | 비고 |
|---|---|---|---|

## 2. 버틀러 Butler — 구조

butler.works · GrowingLab. 현대적 모바일-퍼스트 그래프 기반 기업데이터 제품. "투자비서" 포지셔닝. 4.8★.

- **카테고리**: 펀더멘탈 / 밸류에이션 / 재무 / 주식 / 주주환원 / IR정보.
- **부가 데이터**: 부문별·지역별 실적 · 수출실적 · 수주잔고 · 애널리스트 컨센서스(밸류에이션) · 투자지표.
- **공시 큐레이션(핵심 시그니처)**: 내부자거래·실적속보·배당·자사주·공급계약·시설투자·기업설명회·사업보고서로 *타입 분류* 후 "핵심 정보만 쏙" 요약 카드.
- **부가 surface**: 스크리너(100+ 필터) · 포트폴리오(평가손익/일일손익/평가비중/예상배당금) · 버틀러 매거진(교육).

### Butler의 UX 델타 (차트가 아니라 3가지)
1. **공시 큐레이션 by type.** 우리 disclosure event-rail PRD(`scenario-simulator/11-disclosure-event-rail`)는 *위치*(주가차트 어디)지 *의미별 큐레이션*이 아니다. Butler는 "자사주 취득 공시 떴다"를 타입·요약된 카드로 — 우리는 점 하나. 이게 가장 큰 UX 갭이고 고가치.
2. **세그먼트/지역/수출/수주잔고 데이터.** 세그먼트는 우리 panel에 *존재*하나(`NT_D871100` 축태깅) XBRL 인코딩상 추출 불안정(~2/10). 수출은 관세청=전국 집계라 회사 귀속 불가. → 데이터 경계 = [04](04-data-readiness-kill-list.md).
3. **컨센서스 밸류에이션 + "투자비서" 가이드 + 모바일 청결.** 컨센서스는 DART에 없음(데이터 갭). UX 교훈 = *내러티브-led 진입, 차트 on demand* — 37 차트 벽 아님.

## 3. 두 레퍼런스의 한계 (왜 상위호환이 가능한가)

- **iTooza**: 단일 회사·정적 역사 차트. **peer-relative 없음**(ROE 15%가 동종 상위인지 차트가 답 안 함 → 투자자가 다른 V차트 열어 눈비교). 밸류에이션이 backward-looking(역사 멀티플 밴드지 *가격 함축 미래 기대* 없음). **forensic/품질 층 부재**. 연환산(dcy)이 단순 직전4분기 합산.
- **Butler**: 소비자 등급, *얕다*. 컨센서스 종속(중소형주에서 무력, 컨센서스 편향 미검증). 추천·단정 톤이라 진지한 투자자의 ref-추적 논거에 약함.

→ **둘 다 쓴 뒤에도 투자자는 Excel에서** (a) peer 끌어모아 백분위, (b) 분기 더해 정규화 이익 검산, (c) reverse-DCF로 함축 기대 역산, (d) accruals·재작성·별도연결 괴리 수기 점검 — 을 한다. 이 넷이 우리 빈틈이자 해자(상세 = [02](02-differentiation-killer-features.md)).

## 4. 취사선택 — TAKE / REJECT / DEFER

### TAKE (우리 자산으로 흡수)
- iTooza **주가↔기초체력 지수 오버레이** → gov 주가(2020+) × 기존 panel 리베이스. (SHOULD)
- iTooza **밸류에이션 시계열**(PER/PBR/ROE&PBR) → 밸류에이션 탭 신설 또는 수익성 탭 확장. (SHOULD, post-2020 게이트)
- iTooza **운전자본 회전일수 split** → 기존 CCC 카드 상위호환. (MUST 후보)
- iTooza **3 기간모드(연환산/연간/분기)** → 전역 토글로 기존 카드 배선. (MUST)
- iTooza **업종 인지** → 차트셋 갈아끼우기가 아니라 *카드 가시성 필터*(net 제거). (MUST)
- Butler **공시 큐레이션 by type** → 기존 disclosure source 위 타입 분류·요약 카드. (SHOULD, scenario-simulator 11과 경계 정리)
- Butler **내러티브-led, 차트 on demand** → "기본 뷰 더 작게" 원칙으로 흡수. (MUST 정신)
- 공통 **동종 백분위**(둘 다 약함, 우리만 가능) → `compare` cell-grid. (★SHOULD, killer #1)

### REJECT (복제 금지)
- iTooza 영업외손익률 6선·주당현금흐름·분기배당금(니치/스파게티).
- iTooza 매출지수&순이익지수(주가 없는 버전 — #1 생기면 중복).
- Butler 애널리스트 컨센서스·목표주가(데이터 부재 — 스크래핑은 저작권·환각 유발).
- Butler 포트폴리오·스크리너(다른 제품 surface — 재무분석 랩 범위 밖. 스크리너는 별도 `ScreenerModal` 존재).
- 종합 점수/등급/매수매도 신호(룰 위반).

### DEFER (별도 트랙·차단)
- 세그먼트/지역별 실적(panel 존재, 2/10 clean — 별도 _attempts).
- R&D 추이(_attempts 6/10, 졸업 근접 — Python 게이트).
- 금융업 전용 card set(label drift — sector pull 입증 후).
- 수주잔고·수출 회사매핑(데이터 표면 부재 — BLOCKED/EXCLUDED).
