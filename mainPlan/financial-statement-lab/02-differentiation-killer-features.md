# 02. 차별과 killer 기능 — 왜 우리가 더 나은가

상태: 비전 PRD v0.1 (2026-06-13)
목적: 이 PRD가 *클론이 아닌* 이유를 정량으로 박는다. iTooza·Butler가 구조적으로 못 답하는 질문 5개 + 그 질문을 우리 자산이 답할 수 있는 근거 + 가드레일.

---

## 1. 차별의 핵 — panel cell-grid + `compare`

iTooza는 회사별 사일로다(한 회사 V차트 = 그 회사 데이터만). Butler도 회사 단위 카드다. **둘 다 "이 회사가 동종 분포 어디에 서 있나"를 한 화면에서 답하지 못한다.**

DartLab의 우위는 한 회사를 더 예쁘게 그리는 게 아니라 — **전 회사·전 계정·전 시점이 수평화된 하나의 격자(panel cell-grid)** 라는 데서 나온다. 그 위에:
- `compare(codes, topic, period, scope)` = N사 시점 정렬 비교 엔진. 정렬키 `(dk, scope, leafType)`로 label-drift 자동해소, honest-gap, cross-market guard. (메모리 `project_viewer_peer_compare`, 구현 `panel/compare.py`.)
- reverseDCF(Damodaran식 닻+조건부 범위), credit scorecard, scan/search.

이 격자가 없으면 동종 백분위·가격 함축 기대의 "동종 분포상 상위 몇 %" 둘 다 불가능하다. **이게 1차 해자.** 나머지 killer는 그 위에서만 싸게 돈다.

---

## 2. iTooza·Butler의 분석 천장 (요약, 상세 = [01 §3](01-reference-teardown.md))

- iTooza: 단일회사·정적역사·peer 없음·밸류에이션 backward·forensic 없음.
- Butler: 소비자등급·얕음·컨센서스 종속·추천 톤.
- **공통 결론**: 둘 다 쓴 뒤에도 투자자는 Excel을 켠다. 그 Excel 작업 = 우리 killer 5종.

## 3. Killer 5종 (가치 순위 · 데이터 지원 · 가드레일)

### ★1. 동종 백분위 밴딩 (Peer-relative percentile on every metric)
- **투자자 가치**: "ROE 15%"를 "동종 38사 중 상위 12%, 5년 연속 1사분위"로. 모든 비율 카드에 분포 띠를 깔면 회사의 *체질*이 즉시 보인다. 절대값 → 의사결정값.
- **데이터 지원**: **강함.** `compare` + cell-grid가 정확히 이걸 위해 존재. label-drift 자동해소·cross-market guard·honest-gap이 분포 계산의 함정(결측 0으로 안 깔기, 다른 회계기준 안 섞기)을 *이미* 처리. **1순위인 이유 = 데이터가 가장 완벽히 받치고 경쟁사가 구조적으로 못 함.**
- **가드레일**: 백분위는 **분포지 순위표가 아니다.** "ROE 상위 12%"(사실)는 허용, "종합 1위·A등급"(합성 판정)은 금지. 동종 universe 정의(업종 코드·시총 밴드)는 명시·ref. universe 너무 작으면(<N) honest-gap.

### ★2. 가격 함축 기대 읽기 (Reverse-DCF implied expectations)
- **투자자 가치**: valuation의 성배. "현재가는 향후 10년 매출 9%·영업이익률 14%·ROIC 18%를 요구 → 과거 5년·동종 분포상 상위 5%에서만 가능 → 비싸다." 안커링을 깨고 *price-implied expectation*을 역사·peer 분포에 대조.
- **데이터 지원**: **강함.** reverseDCF 엔진 존재 + panel이 과거 마진·ROIC 분포 + `compare`가 "요구치가 동종 상위 몇 %인가" 즉답. #1과 결합 시 *함축 기대를 분포에 꽂는* 독보 조합.
- **가드레일**(★중요): **단일 적정주가·목표주가 금지.** 출력은 "가격이 요구하는 것 vs 회사가 해온 것 vs 동종이 하는 것"의 *읽기*지 "적정주가 = X원"이 아니다. WACC·terminal 가정은 블랙박스 금지·honest 노출. **이건 scenario-simulator [08-valuation-report](../scenario-simulator/08-valuation-report.md)의 "reverseDCF 닻+조건부 범위" 패턴과 동형** — 단 이 랩은 *정적 읽기*(asOf 현재가 기준), 시뮬레이터는 *미래 what-if*. 경계: 미래 가정 토글·재생버튼은 이 랩에 없다(그건 시뮬레이터). 이 랩은 "지금 가격이 이미 가정한 것"만 역산.

### 3. 이익 품질 / accruals / 이익의 현금화 forensic 플래그
- **투자자 가치**: "이익이 진짜 현금인가"의 자동 1차 방어. CFO/NI 추세, 발생액(ΔNWC/매출) 누적, 매출채권·재고가 매출보다 빨리 느는가, 영업이익↔CFO 괴리 확대. 분식·품질 악화 조기경보.
- **데이터 지원**: **중상.** panel에 IS/BS/CF 전 계정·다년 → accruals·CFO/NI·운전자본 회전 전부 계산 가능. *솔직한 한계*: 일회성·특수관계자·재작성 이력은 정기 패널 외 공시(주석·정정)에 의존 → scan/search로 끌어오나 "정량 플래그"는 가능, "확정 분식 판정"은 아님.
- **가드레일**: **경보지 단정 아님.** "CFO/NI 1배 미만 3년 지속" = 사실 surface, "분식이다" = 금지. cardGuide.bad에 이미 적힌 일반 임계(`cashConversion`: "1배 미만 지속 = 분식의 고전적 경고등")를 데이터 위에 *배지*로 — 새 주장 0.

### 4. Cross-statement 정합성 체크 (tie-out)
- **투자자 가치**: "숫자를 믿어도 되나"의 기계적 검산. IS 순이익 → CF 출발점 일치, BS 이익잉여금 증분 = 순이익−배당, OCI bridge가 자본변동표와 맞물리나, 별도 vs 연결 괴리 추세. 투자자가 손으로 하던 tie-out 자동화.
- **데이터 지원**: **강함.** 이미 OCI bridge·CIS→IS fallback·연결/별도 scope 처리 보유. honesty spine("결손 0 대체 금지")이 정합성 검사의 전제. **남이 안 하는 이유 = 지루하고 정합이 어려워서 — 우리가 이미 풀어둠.**
- **가드레일**: 불일치는 "점검 필요" flag지 "조작" 단정 아님. 정상 사유(연결범위 변경·비지배지분·환산차이) 존재 → flag는 *주의 환기*지 *판결*이 아님.

### 5. 올바른 TTM·연환산 + 일회성 정규화 이익
- **투자자 가치**: iTooza dcy의 상위호환. 계절성·연결범위·인수 타이밍 반영한 진짜 TTM + *일회성 제거 정규화 이익*으로 멀티플 분모를 있는 그대로 표시. "PER 싸 보였는데 일회성 영업외이익 때문"을 걸러냄.
- **데이터 지원**: **중상.** TTM·분기규약 자동판정·CIS→IS fallback 보유 → 기계적 TTM 강함. 일회성 정규화는 주석 파싱 의존 → *부분 자동 + 라벨링*(어디까지 정규화했는지 ref 노출).
- **가드레일**: 정규화 범위 명시·ref. "정규화 이익"을 공식 fact처럼 내세우지 말고 "공시 이익 + 우리가 제거한 항목" 둘 다 보여줌.

## 4. 우선순위 종합 (value-per-effort)

| 순위 | killer | 데이터 | 단계 |
|---|---|---|---|
| 1 | 동종 백분위 밴딩 | 강 (`compare` 완비) | SHOULD (배선) |
| 2 | 운전자본 회전일수 split | 강 (panel) | **MUST** (싼 업그레이드) |
| 3 | 가격↔기초체력 지수 오버레이 | 중 (gov 주가 2020+) | SHOULD |
| 4 | 이익품질 forensic 플래그 | 중상 (panel + 주석) | SHOULD |
| 5 | cross-statement 정합성 | 강 (보유) | SHOULD |
| 6 | reverseDCF 함축 기대 읽기 | 강 (엔진+compare) | SHOULD (가드레일 엄격) |
| 7 | 정규화 이익 | 중상 (주석 의존) | DEFER (Python 게이트) |

투자자가 *즉시 체감*하는 4: #1(동종 어디?) · #6(비싼가?) · #3(가격 vs 가치?) · #3+4(믿어도 되나?).

## 5. "왜 iTooza 안 쓰고 DartLab" — 4문장 피치

iTooza·Butler는 한 회사의 역사를 *그려주고* 거기서 멈춘다. DartLab은 같은 차트 위에서 셋을 더 *답한다* — 이 회사가 동종 38사 분포의 어디에 서 있는지(절대값이 아니라 백분위), 현재 주가가 어떤 성장·마진·ROIC를 *요구*하는지(역사 멀티플이 아니라 함축 기대), 이 숫자를 믿어도 되는지(이익의 현금화·정합·일회성). 그리고 그 모든 숫자는 추정이 아니라 원천 공시로 역추적되고, 결손은 0으로 채우지 않으며, 추천하지 않는다 — 당신이 자기 논거를 세우라고. **iTooza는 당신이 차트를 보고도 Excel을 켜게 만들고, DartLab은 그 Excel 작업을 답으로 바꾼다.**
