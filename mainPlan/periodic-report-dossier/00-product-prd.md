# 00 — 제품 PRD (정본)

## 1. 무게중심 (한 문장)

비재무 사업보고서 실체 — dartlab 이 이미 네 번이나 fetch 하면서도 얕게·출처 없이 흘려보내는 **사람·자본배분·소유** 팩트 — 를 **하나의 날짜 박힌, 출처추적되는, 자기이력 프레임의 도시에** 로 묶되, **패널 수는 줄이면서** 가독성과 정직성을 올린다.

## 2. 비전

> 한국 기업의 사업보고서를 *하나의 날짜 박힌 클릭 가능한 문서* 로 읽는다 — 누가 일하는지(인력·생산성), 주주가 실제로 무엇을 받는지(소각 vs 금고), 자본이 어디로 흘렀고 누가 지배하는지(타법인출자·소유) — 모든 숫자는 그 DART 공시(`rcept_no`)로 한 클릭에 검증되고, **자기이력 + 유니버스 순위** 로 의미를 얻으며, **결코 매수/판정으로 환산되지 않는다**.

이 PRD 가 소유하는 고유 영토 = **재무제표가 아닌 정기보고서 섹션들** — 손익/재무상태/현금흐름표는 financial-statement-lab 이 소유. 우리는 그 *바깥*, 사업보고서의 인적/운영/자본배분/소유 현실을 소유한다.

## 3. 문제 (사용자 명령의 정밀 진단)

사용자: "정기보고서 팩트부터 그아래 인력, 생산성 등 주주환원까지 타법인 출자 위까지 더 강화 … 우리는 강력한 정보들이 있다 그걸 다 못쓰고 있다."

실측 진단:
- **수집 28 / 차트화 14**. 절반(major-shareholding 변동·자금조달 사용·임원보수 거버넌스 분리·하이브리드자본 등)이 수집만 되고 방치.
- **노출된 것조차 얕다**:
  - 직원 — `report.workforce()` 가 전 연도 `WorkforceYear[]` 를 주는데 우측 패널은 `wfLast`(최신 1년) 6개 평면 팩트만. 급여 상위3명만 보임.
  - 주주환원 — `buybackCancel`(소각, 진짜 vs 화장 환원을 가르는 단 하나의 필드)을 fetch 한 뒤 버린다. 배당 수익률만 강조.
  - 타법인출자 — lossPct(적자 피출자 점유)·control-shift(지배 이동)가 `HoldingsDialog` 안에 갇혀 한 클릭 뒤에만 보임.
- **중복 표면화**: 같은 dividend/treasury parquet 가 `DART 정기보고서 팩트` 평면 패널 + `주주환원` 패널 양쪽에 그려진다.
- **출처·정직 라벨 부재**: `loadLiveCompanyReportFacts` 가 `rcept_no` 를 SELECT 에서 버려, 어떤 팩트도 원문 공시로 클릭 이동 불가. 미공시는 `—` 로 표시되긴 하나 as-of 지연·커버리지 N 이 안 보인다.

핵심 통찰: **데이터는 있다. 못 쓰는 건 "표면화의 얕음 + 출처의 부재 + 의미층의 부재"** 다. 그래서 해법은 *새 분석이 아니라 갇힌 계산의 정직한 표면화*.

## 4. 차별화 (왜 다른 데선 못 얻나)

아이투자·증권사 HTS 는 배당수익률과 평면 배당이력을 보여준다. 다음 3개는 *이미 fetch 된 데이터에서 한 join 거리*에 있고, 리테일 도구엔 거의 없으며, **한국 시장에서만 압도적**이다:

1. **소각 vs 금고** — 진짜(영구 환원) vs 화장(재매각 가능·잠재 희석) 자사주. 밸류업/코리아 디스카운트가 정면으로 겨눈 질문이자 이 PRD 가 전면에 올리는 단 하나의 필드(`buybackCancel`).
2. **재벌/지주 순환출자 소유 웹** — 타법인출자(자본 out) + 최대주주(지배 in)를 한 종목코드 공간에서 교차. `HoldingsDialog` 에 *이미 세계급으로* 구현됨 → lossPct(부실 계열사에 묶인 자본) + control-shift 로 한 클릭 앞으로.
3. **인적자본 풍부함** — 정규/계약·근속·성별·가중평균급여는 한국 사업보고서 의무 필드(미국 10-K 에 거의 없음). 회사 vs 유니버스 인적자본 효율 순위 = 한국 고유, 글로벌 복제 불가.

더 깊은 해자: **dartlab 이 이미 만들어 묻어둔 엔진들** (HoldingsDialog 관계망, `scanValueAdded`, `calcRndExpense`, 백분위 머신). 강함은 새 분석이 아니라 — *갇힌 계산을 한 클릭 앞·정직 라벨과 함께 표면화*. 어떤 리테일 도구도 비재무 사업보고서를 *하나의 날짜 박힌 탐색 가능한 문서* 로 읽지 않는다.

## 5. NEVER-CLAIM (정직 가드 — 위반 시 제품 폐기 수준)

1. 비재무 팩트 위에 **종합 등급/점수/레이더 금지** (거버넌스 점수·좋은 고용주 등급·운영건강 평점 없음).
2. **매수/매도/판정/"주주친화 우량주" 톤 금지** — 측정 + as-of + 출처만, 판단은 사용자.
3. **"소각=무조건 호재" 금지**(전환사채 헤지 소각 맥락 존재); **"계약직 비중 낮음=고용안정" 금지**(부외 외주 가능).
4. **동종 백분위 금지** — 단, 인적자본 유니버스-rank 축 1개 예외(peer-relative 백분위 = financial-statement-lab 영토).
5. **시계열/궤적 백분위 금지** — 단일시점만(궤적 = scenario-simulator 경계).
6. **가동률/생산능력을 숫자·추세선·차트로 금지** — narrative 원문 발췌만, 절대 수치화 안 함.
7. **결측 0-fill·impute 금지** — `—`/미공시/해당없음/부분 을 first-class 상태로, 절대 0 아님.
8. **시총분모 총주주환원율 금지**(gov price T+1·2020+ = stale·거짓정밀); contribShare 에 mktcap/PER 함축 순익 분모 금지(보고 `fin.is.net` 쓰거나 라인 삭제).
9. **자사주 소각 "가치" 를 raw 원 단위로 금지**(취득원가 컬럼 없음 — 추정=주식수×종가 라벨 또는 생략).
10. **2-cycle 상호출자 K건 → "순환출자=K" 단정 금지**(다단 순환 미탐지); 백분위 sector-adjusted 단정 금지(실제 그럴 때만).
11. **질권/담보 claim 금지** — gather 데이터에 필드 없음. 침묵.
12. **우측 레일에 그래프 금지**(스파크라인·흐름막대 없음) — 그래프는 center-stack FinFullscreen 으로만.

## 6. 성공 기준

- 순 패널 수 **감소**(평면 팩트 패널 → 리본 흡수, 섹션은 기존 패널 재사용).
- 모든 비재무 팩트가 `↗원문` 으로 정확한 DART 공시에 한 클릭 도달.
- 3개 간판 리프레임(소각vs금고·lossPct/control-shift·인적자본 자기이력)이 **새 fetch 0** 으로 동작.
- 데모 게이트: 3/6 커버리지 소형주 + 무배당 소형주 + 첫배당 케이스가 *삼성 아닌* 정직 빈상태로 렌더(전상장사 커버 증명).
- NEVER-CLAIM grep 가드 green. svelte-check 0. 공개/로컬 동일 렌더.
