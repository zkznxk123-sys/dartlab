# 03 — 정보구조 + UI/UX (전달 매커니즘)

> 사용자 핵심 요구: "사용자들에게 정보를 잘 전달할 매커니즘". UI/UX 전문가 2인(정보구조 + 의미층) 설계.

## 1. IA 결정 — 분산(distributed) + 스파인(spined)

**통합 "기업 실체 / Company Reality" 6번째 레일 = 기각.** 3가지 근거:
1. 정확히 `feedback_always_check_clutter` 가 금지하는 덕지덕지 누적(6번째 레일).
2. RightStack 이 이미 1회 fetch 하는 상태(`wf[]`/`srs[]`/`inv[]`/`shPeriods[]`)를 중복하거나 2번째 fetch 경로 fork(공통배선·dedup 위반).
3. **FinFullscreen 이 이미 progressive-tab 깊은 surface** — 2번째 탭 셸이 그것과 경쟁.

**순수 분산(각 패널에 줄만 추가) = 불충분.** 평면 중복 패널과 `rcept_no` 출처추적 부재가 남음.

**채택 = 스파인이 분산을 도시에로 묶되 통합 셸은 안 짓는다.** 3 구조 수:
1. **스파인/헤더**: 기존 `DART 정기보고서 팩트` 패널(L520) → 한 줄 도시에 헤더 리본. 평면 6행 덤프가 리본으로 **붕괴**(중복 1패널 삭제).
2. **섹션**: 4 기존 패널(인력 L541 / 주주환원 L555 / 타법인출자 L569 / 거버넌스 L745)을 레일 순서 그대로 *도시에 섹션*으로 — 각자 ① 같은 공시로의 ↗ ② 환원불가능한 자기정규화 문장 1줄. *이동 아님, 시각 처리(공유 공시 앵커)로 섹션화.*
3. **깊이**: 각 섹션 `상세보기` → **기존** 깊은 surface(인력→PEOPLE 탭[누락 `상세보기` 추가], 주주환원→RETURN 탭[확장, 신규 탭 금지], 타법인출자→HoldingsDialog[불변]). 백분위 축은 **기존** 백분위 머신/PercentileCrossDialog 로.

**정합성 증명: 순 패널 Δ = −1.** 레일이 *짧아지고* 스파인을 얻는다. 읽기 순서가 위→아래로 이야기: 누구인가(공시 앵커) → 누가 일하나(인력) → 주주가 받는 것(주주환원) → 자본이 흐른 곳·지배(타법인출자·소유). center/left 레일 불변.

## 2. 전달 매커니즘 — 3 가독성 층 (위젯 아님, 균일 적용)

1. **공시 앵커(크기보다 최신성)**: 비전문가는 "배당총액 500억" 못 읽지만 "기준 2024.12, 접수 2025.03" 은 즉시 읽는다. 리본이 전 스택에 as-of 1회 스탬프 + ↗ 로 실제 공시. **↗ 가 가장 중요한 전달 행위 — 모든 숫자를 한 클릭에 반증 가능하게.**
2. **섹션당 자기정규화 문장 1개(raw count 아님)**:
   - 인력: "1인당 부가가치 상위 18% · 급여 6%p 빠르게(매출 대비)"
   - 주주환원: "취득의 90%가 소각 · 발행주식수 −3.2% · 12년 연속배당" → "내 지분이 가만히 있어도 3.2% 늘었다"
   - 타법인출자: "적자 피출자사가 출자 장부가의 30% 점유" + "최대주주측 51%→(Δ)"
   - R&D: "연구개발 집약도 6.8% ↑ (IS)" (인력 격자 1행, 텍스트 추세)
3. **진실로의 점진 ↗**: 스냅샷 문장(레일) → 상세보기(FinFullscreen 탭/HoldingsDialog) → ↗원문(DART 공시 자체). 3 고도, 각 1클릭, 전부 기존 surface 재사용.

## 3. 점진 공개 (SNAPSHOT → DETAIL → COMPARE)

- **SNAPSHOT(0클릭, 우측 레일)**: 도시에 리본 + 섹션당 문장 1줄 + raw 격자(불변). 커버리지 라벨(N/6·미공시) first-class.
- **DETAIL — 글로서리(1클릭 `!`)**: CARD_GUIDE 팝오버(MiniFinChart `.mfcGuide` 패턴: 읽는법/올라가면/내려가면/foot-caveat). 순수 교육, 회사별 판정 0.
- **DETAIL — 도시에(1클릭 `상세보기`)**: FinFullscreen 탭/HoldingsDialog — 전 자기이력 시계열 + 갇힌 계산. 환원흐름 막대는 여기(center-stack, 그래프 합법).
- **COMPARE(1클릭, 백분위 상세보기)**: PercentileCrossDialog — *유일한* 동종 랭킹 surface, '분포 사실, 판정 아님' 으로 fence. 인적자본 1행. sector-relative 명시 defer/라벨.

각 고도는 *자기 caveat 가 지지하는 비교만* 표시: 스냅샷은 백분위 안 보임(다이얼로그 caveat 필요), 다이얼로그는 시계열-백분위 궤적 안 보임(경계 회귀, 컷).

## 4. 의미층 거버넌스 — `올라가면`/`내려가면` 리프레임 (전체 핵심)

CARD_GUIDE 의 `good`/`bad` 필드는 문자 그대로 '긍정/부정'. 신규 비재무 키는 **'good' 이 사용자 판단인 지표에서 `good`/`bad` → `올라가면`/`내려가면`(서술적 읽기)로 리프레임**, 절대 처방 아님. 서술-only 필수 예:
- 계약직 비중 낮음 (부외 외주 가능, "고용안정" 아님)
- 자사주 소각 (CB 헤지 가능, "무조건 좋음" 아님)
- 급여매출괴리 (보편 good 방향 없음 → DistCurve neutral 그레이핀)

`good`/`bad` 유지는 구조적으로 명백한 방향만(R&D 집약도 상승=재투자는 사실, 단 가이드는 '미래 재투자' 라 쓰지 '우량' 아님).

## 7. UI/UX 리스크

1. **헤더 회귀**: 평면 패널 붕괴가 일부 팩트 누락 위험 → dividend/treasury 는 주주환원 섹션에 존재(중복만 제거). 삭제 전 모든 reportFacts 키 재표현 확인.
2. **good/bad → 올라가면/내려가면 리프레임이 전체 승부**: 주니어가 기존 '우량/위험' 톤 복사 시 verdict 위반 → cardGuide.ts 상단 convention 주석 + G3-style grep 가드 확장(우량/매수/좋은고용주/주주친화 금지) + 리뷰어 신규 가이드 문자열 낭독.
3. **자기이력 Δ 전년 부재**(첫배당 소형주) → 토큰은 prior 값 non-null 일 때만 렌더(`—→61%` 아닌 `61%`).
4. **백분위-자기이력 고도 혼선**: 'top 18% AND ↑3년전' 한 행 = 시계열 백분위 재수입(KILL) → 스냅샷 행은 백분위 *또는* 자기 Δ, 동일 지표 둘 다 금지.
5. **레이아웃법 누출**: 흐름막대/궤적을 레일에 두려는 유혹 → 그래프=center-stack only, 백분위=단일시점. 카피는 NEVER-CLAIM grep 게이트.
6. **도시에 as-of 상속이 거짓 될 수**: 인력=연 단위라 리본 2024.12 인데 인력 2023 → 섹션은 자기 연도를 `sub` 에 유지(이미 '임직원 2024'), 리본=공시 앵커·sub=섹션 as-of, 둘 다 표시.
7. **전상장사 커버 검증**: 데모 = 3/6 소형주 + 무배당 + 첫배당(삼성 아님), 빈상태 first-class 산출물. push 전 공개+로컬 동일 렌더 검증(UI push 운영자 게이트).
