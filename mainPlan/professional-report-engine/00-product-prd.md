# 00 · PRD — 전문 애널리스트 리포트

> 출처: 전문 에이전트 설계 4종(주식 애널리스트·신용 애널리스트·능력 검증가·서사 설계가) 종합. 본 문서는 *리포트가 무엇인가*를 박는다 — 능력 엔진 빌드(02)와 엔진 갈아엎기(03)는 이 비전을 향해 수렴한다.

## 1. 한 줄 정의

같은 회사를 **하나의 인과 논증(thesis-led causal arc)** 으로 위→아래로 검증하는, 콜(call)을 내는 전문 리포트. "5관점 병렬 나열"도 "측정값 카탈로그"도 아니다.

## 2. 핵심 진단 — 약한 건 조립층이지 엔진 *기반*이 아니다

dartlab 엔진은 이미 DCF/DDM/RIM·신용 20등급·민감도 격자·세그먼트 주석추출을 **갖췄다**(단 일부는 아마추어 수준·미배선 — 01·02 에서 격상). 약한 건:
- thesis 가 정규식으로 긁은 기계 문장.
- 5관점이 서로 충돌·강화하는 *긴장*이 안 드러나 "5장(章) 나열"로 읽힘.
- 밸류에이션·전망이 과거 외삽 수준(02 에서 프로 격상).

→ 작업 본질 = *새 데이터 창조*가 아니라 **강화된 전문가급 출력을 처음으로 논증으로 엮기.**

## 3. 서사 아크 (순서가 곧 논증)

```
[0] 결론 먼저 (Thesis)   중심논거 + 검증가능 지지기둥 3 + 약세론(논쟁) 1 + 관점전환 트리거
[1] 사업구조   무엇 팔아 버나       매출동인·세그먼트
[2] 수익체력   마진 어디서 갈리나    OPM 브리지·ROIC vs WACC
[3] 현금전환   이익이 현금인가       CFO/NI·CCC·FCF
[4] 자본배분   현금 어디 쓰나        CF 워터폴·재투자 회수율
[5] 재무·신용  버티는가             레버리지·ICR·dCR 20등급·만기사다리
[6] 경쟁위치   업종 어디쯤·해자      peer 백분위·정량 moat·life-cycle
[7] 밸류에이션 얼마짜리인가          내재가치 콜 + 시나리오 + reverse-DCF
[8] 포워드뷰   다음 장은            driver 기반 전망(백테스트된) + 매크로
[9] 리스크     뭐가 틀리나          bear case·신용 하방 시나리오
[10] 종합·근거 정리 + 증거 부록(ref)
```

각 섹션은 "→ 그래서"로 다음 섹션의 질문을 연다(`transition`). 가치 = 현금흐름의 현재가치이므로 매출→마진→현금→배분→수익률(ROIC>WACC) 사슬을 따라가야 밸류에이션이 의미를 갖는다. 밸류를 사업 이해 앞에 놓으면 점쟁이.

## 4. Thesis 규율 — 기계 문장 폐기, 구조화 논증

```
THESIS (최상단, 본문 finding 에서 합성되어 앞에 배치)
├─ 중심논거   검증가능 인과 1문장 (형용사 아닌 메커니즘): "ROIC 18%로 WACC 6%p 상회 → 자본이 가치 창출"
├─ 지지기둥 3  각 기둥 = 본문 섹션 1개로 증명 + evidenceRef 결박
├─ 약세론(논쟁) "이 논지를 깨는 단 하나는 X" — bear case 가 thesis 와 동등 무게
└─ 관점전환 트리거 "다음을 보면 수정: ① CCC 2분기 급증 ② ICR 1배 하회 ③ ..."
```

전문성의 핵심 = **반증 가능성**("좋다" 아닌 "ROIC 18%>WACC 12%") + **약세론을 진지하게** + **콜을 낸다**(범위 뒤에 안 숨음).

## 5. 밸류에이션 — 콜을 내는 설계 (상세 = 02)

- **회사별 bottom-up WACC** (섹터 평균 폐기): CAPM(re-levered 베타 + macro 무위험금리 + 국가 ERP) + **dCR 20등급 → 부채비용 스프레드**(신용엔진 직결).
- **재투자 묶인 성장**(Damodaran 항등식): g = 재투자율 × ROIC. 과거 CAGR 폐기. ROIC 를 명시구간 동안 WACC 로 fade.
- **내재가치 콜 + 시나리오**(bear/base/bull 은 세 드라이버 — 성장·마진·WACC — 를 움직여 생성, ±10% 장난 아님).
- **Reverse-DCF**(결정타): 현재가가 내재한 성장률 역산 vs 펀더멘탈 지지 성장률 → "시장은 12% 가격에 넣음, 지지는 7% → N% 고평가". 미래 예측 아닌 *시장 가정 해독*.

## 6. 신용·리스크 — 얇은 "재무안정성" 렌즈를 밀도 높은 신용 섹션으로 대체 (상세 = 02·E)

- dCR 20등급 + forward PD 를 **라이브 배선**(최고 ROI 차별화, 79사 검증).
- 7축 가중 워터폴로 "등급을 증거로 벌었음" 시각화 + divergence(신평사 notch 차이) + 궤적.
- bear case = *과거 실제 최악 낙폭 재현 + 만기 스케줄(확정) + 결정론 산식*. 미래 점추정·부도 단정 금지(이건 날조 금지 선이지 honest-skip 이 아님).
- 포렌식 플래그 = 신호이지 판결 아님(수량/severity 표기, "감지" O · "유죄" X).

## 7. 서사 문법 — 블록 어휘 8 → 18, 계약 SSOT (상세 = 03)

`ui/packages/contracts/src/report.ts` 로 ReportModel + 블록 어휘 승격. Python·TS 둘 다 conform. 신규 10블록:
`thesis · exhibit(번호·제목·테이크캡션·source·단위) · callout · verdict(noComposite:true 합성점수 영구차단) · scenario · valuationBridge · peerScatter · driverTree · excerpt · transition`. 차트 전부 MiniFinChart SSOT 위임.

## 8. 프로 vs 아마추어 기준 (수용 게이트)

1. Thesis-first(결론 먼저), not data-first.
2. 모든 주장에 evidenceRef(재무제표 라인·rceptNo) 결박.
3. 약세론을 정량으로(valuationSins·creditScenario) 동등 무게.
4. 콜을 낸다 — 내재가치·신용등급·상대위치. 범위·"투자조언 아님" 뒤에 숨지 않음(단 가정·민감도는 노출).
5. 포워드 동인 정량화(백테스트된 전망), 막연한 "성장할 것" 금지.
6. 세그먼트·moat 를 *모델로* 산출(미공시여도 도출 방법 명시), 게으른 스킵 금지.
7. 인과 연쇄로 섹션 연결.
8. 미검증 확신 금지 — 모델은 백테스트/민감도로 증명 후 탑재.

## 9. 비목표 (out of scope)

- sell-side 컨센서스 수집(dartlab 미수집 — 자체 driver 전망으로 대체).
- 정성 환각 moat 서술(02·D 에서 *정량* moat 로만).
