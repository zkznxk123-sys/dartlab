# 00. Product PRD — 공시 테이블 엑셀 내보내기

상태: v0.1 (개정 이력은 06 원장)
범위: DartLab 공개 UI·로컬 터미널·엔진의 "공시 테이블 → 엑셀" 제품 목표·경험·성공 기준

---

## 1. 배경 — 사용자 문제

회계·IR·리서치 실무자는 공시(사업보고서·분기보고서)의 표를 **엑셀로 옮겨 직접 계산**한다. 현재 그 과정은:

1. DART 원문에서 표를 찾아 드래그 복사 → 엑셀 붙여넣기 → 병합셀·단위·괄호음수가 깨져 수작업 재정리.
2. 여러 기간을 보려면 보고서를 연도별로 따로 열어 표를 매번 다시 복사.
3. 여러 회사를 비교하려면 회사마다 같은 작업을 반복.

참조 외부 서비스가 푼 부분은 **"공시의 테이블을 통째로 엑셀로 빼주기"** 다 — 미리보기에서 표를 보고, 클릭해 내보내기 양식을 만들고, 그 양식으로 한 번에 내보낸다. 이 개념은 명확히 유효하다.

그러나 그 서비스는 **공시 1건의 표를 그 시점 구조 그대로** 빼주는 데 머문다. DartLab은 이미 **panel** — 공시 항목이 `disclosureKey/sectionKey`로 canonical 정렬돼 **여러 기간이 한 행에 수평 정렬**되고 **회사 간에도 같은 키로 비교 가능**한 격자 — 를 가지고 있다. 이 자산을 레버로 삼으면 같은 "표 내보내기"를 **다기간 시계열**·**회사 간 일괄**로 확장할 수 있다. 외부 서비스가 원천적으로 못 하는 영역이다.

> 본 PRD의 명제: *외부 서비스의 "미리보기→선택→양식→내보내기"를 1:1 흡수하되, 우리 panel로 그들이 못 하는 2가지(시계열·회사이식)를 최소 UI로 얹어 차별화가 즉시 보이게 한다.*

---

## 2. 제품 목표

1. 한 회사 공시의 **모든 테이블**을 엑셀로 내보낼 수 있다. 재무제표뿐 아니라 주석·지배구조·사업 등 공시에 나오는 표 전부.
2. 사용자는 **뷰어 미리보기에서 표를 보고 클릭해 선택**한다 — 별도 화면 전환 없이.
3. 선택은 **내보내기 양식(template)**으로 저장·재사용된다. 한 번 만든 양식을 다음에, 그리고 **다른 회사에** 다시 쓸 수 있다.
4. 내보낸 엑셀은 **바로 계산 가능**하다 — 숫자는 진짜 숫자(콤마 텍스트 아님), 음수는 음수, 병합셀은 보존, 단위는 헤더에 명시.
5. **목차(TOC)가 시트 구조**가 된다 — 사용자가 미리보기에서 본 구조 그대로 시트로 펼쳐진다(WYSIWYG).
6. 3 표면 — 엔진(Python, 권위 .xlsx), GitHub Pages 공개(브라우저 .xlsx), 로컬 터미널 — 에서 **같은 선택 인터랙션**으로 동작한다. 산출 백엔드만 다르다.
7. 공개면은 로컬의 상위 충실도(스타일·자동너비)를 *숨기지 않고* tier 표시 + 업그레이드 hint로 보인다.

## 3. 차별화 — "더 좋은 개념" (채택/기각 결정표)

| # | 개념 | 판정 | 근거 |
|---|---|---|---|
| **a** | **테이블당 2모드: as-filed ↔ horizontalized** | **채택 — 1순위** | 외부 서비스 **원천 불가**. as-filed = 선택 기간 1개의 공시 원본 구조(병합 보존). horizontalized = 같은 표를 항목×전기간 시계열로(panel `PanelRow.cells: Record<period,xml>`가 이미 보유). 토글 1개. 데이터는 이미 격자에 있어 **추가 비용 0**. |
| **b** | **회사 간 양식 이식** | **채택 — 2순위** | 양식의 `sectionKey/disclosureKey` 집합을 N개 회사에 그대로 적용 → 같은 표를 N사 파일로. `dartlab.compare(codes, topic, period, scope)` + `buildCompareBoard`(disclosureKey 정렬 보장, 이미 작동)가 받침. 외부 서비스 불가. |
| **c** | **재무=숫자 DataFrame / 공시=raw 구조 자동 판정** | **부분 채택 — 자동, 토글 없음** | finance 정량(IS/BS/CF/CIS)이면 깨끗한 숫자표(`readStatement`/`financeToExcel` 경로), 일반 공시 표면 as-filed raw 구조. "raw냐 clean이냐"를 사용자에게 묻지 않는다(덕지덕지). |
| **d** | **provenance + honest-gap** | **채택 — 기본 ON, 끌 수 있음** | 각 시트(또는 별도 `_출처` 시트)에 회사/종목코드/rceptNo/period/dartUrl/asOf/가공표기. 결손은 빈 셀(0 대체 금지). `dartUrlByPeriod`·`rceptNo`는 이미 bundle/leaf에 있음. |

### 3.1 명시적으로 하지 않는 것 (덕지덕지 차단)

- ❌ **3번째 이상의 모드**(diff·정규화 모드 등). 2모드만. 강함은 깎아서.
- ❌ **수식 자동 주입**(`=SUM()` 삽입). 데이터 충실도만 보장하고 계산은 사용자에게. 수식 생성은 검증 부채.
- ❌ **차트 자동 삽입**. 내보내기는 데이터지 비주얼이 아님(차트는 별 기능 `CompileVisual`).
- ❌ **셀 단위 raw/clean 수동 선택 UI**. (c)를 자동 판정으로 흡수했으므로 토글 노출 안 함.
- ❌ **horizontalized 기본 강제**. "공시 원본 그대로"(as-filed)가 사용자 멘탈모델 기본값. 수평화는 토글 우측 힌트로만 발견.

## 4. 범위 / 비범위

### 4.1 범위
- DART(KR) 공시 panel의 모든 `blockType==='table'` 행. 텍스트(서술) 블록도 선택 시 셀 내 줄바꿈 보존해 시트로 가능.
- EDGAR(US)도 같은 panel 격자(`market='US'`)라 동일 경로 — 단 1차 검증은 DART, US 확장은 후속.
- 엔진 .xlsx(openpyxl), 브라우저 .xlsx(zero-dep OOXML), 양식 CRUD(로컬 파일/서버 + 브라우저 localStorage).

### 4.2 비범위 (이번 라인)
- 워드/PDF/HWP 내보내기. (엑셀만.)
- 엑셀 → DartLab 역(import). 단방향.
- AI가 "삼성 손익이랑 배당 엑셀로 빼줘"를 selection으로 변환하는 자연어 경로 — 아키텍처는 `viewerActions` 채널이 받게 설계하되(03 §5) 구현은 후속.
- 차트·수식·피벗테이블 자동 생성(§3.1).

## 5. 핵심 사용자 흐름

### 5.1 단일 회사 (주 경로)
1. 뷰어에서 회사를 연다(이미 panel·TOC 로드됨).
2. 헤더 `[표 내보내기]` 클릭 → 선택 모드 on. 격자 셀과 TOC 항목에 체크박스가 fade-in, 우측 `ExportDrawer` 열림.
3. 격자에서 원하는 표(셀)를 체크하거나 TOC에서 섹션을 통째 체크 → 드로어 바구니에 시트로 추가.
4. (선택) 드로어에서 시트 순서 드래그·시트명 편집·모드 토글(원본↔수평)·기간 좁히기.
5. `[내보내기 ⬇]` → 표면별 산출(공개=브라우저 .xlsx 즉시 / 로컬=엔진 .xlsx). 다운로드 토스트.

### 5.2 회사 간 이식
- 드로어 `[회사 이식 ▸]`에서 회사 검색(비교모드 `CompanySearch` 재사용) → 같은 양식을 N사에 적용. 로컬은 회사당 .xlsx + ZIP, 공개는 LRU 캐시 내 회사로 제한 + 그 이상은 로컬 안내.

### 5.3 양식 재사용
- `[양식으로 저장]` → `ExcelTemplate`(JSON). 다음 세션/다른 회사에서 양식을 골라 즉시 내보내기. 로컬은 `~/.dartlab/templates` + 서버 CRUD, 공개는 localStorage + 동봉 프리셋.

## 6. 성공 기준

제품 성공:
1. 사용자가 공시 뷰어에서 표를 클릭만으로 골라 엑셀로 받는다 — DART 원문 복붙 없이.
2. 받은 엑셀에서 즉시 합계·증감을 계산할 수 있다(숫자 타입·음수·단위·병합 보존).
3. 한 표를 "원본 1기간"과 "5년 시계열" 양쪽으로 받을 수 있다(외부 서비스 대비 차별화 체감).
4. 한 번 만든 양식을 다른 회사에 적용해 N사 파일을 한 번에 받는다.
5. 공개 사용자도 같은 인터랙션을 쓰고, 로컬 완전판으로의 업그레이드 지점을 본다(funnel).

엔지니어링 성공:
1. 신규 코드가 기존 `viz/export/`·`dataExport.ts`·`cell.ts`·뷰어 컴포넌트의 **확장**이며 fork·중복이 없다.
2. 엔진·브라우저 격자 파서가 **같은 골든 픽스처**를 통과한다(패리티 게이트).
3. 같은 `ExcelTemplate` JSON이 public·local에서 동일 시트 구조 .xlsx를 낸다.
4. 브라우저 번들에 xlsx 라이브러리가 추가되지 않는다(zero-dep 유지).
5. 일괄 내보내기가 OOM 강행규칙(동시 ≤2, 회사당 flush)을 위반하지 않는다.

## 7. 개발자 · PM 이중 평가

- **개발자**: 진짜 신규는 3개뿐 — ① 정합 정규화(숫자/음수/단위/격자, `tableExtract.ts`+엔진 대응) ② source 모델 union 확장 ③ 브라우저 zero-dep OOXML writer. 나머지는 배선: 미리보기=기존 격자, 드로어=AskDrawer 슬롯, 선택 glow=`data-cell` 키 재사용, 회사이식=`?vs=`/`compare` 인프라, 시트명 트림=기존 `sheetName`/`excel.py`, 템플릿 CRUD=기존 `TemplateStore`+서버. 리스크는 zero-dep OOXML zip 구현 정확도(§02)와 엔진↔브라우저 파서 발산(§02 §3)이며 둘 다 골든 픽스처로 통제.
- **PM**: 외부 벤치마크의 핵심 UX를 1:1 흡수하면서, 그들이 절대 못 하는 2개(시계열·회사이식)를 토글/버튼 각 1개로 얹어 "왜 DartLab인가"가 첫 사용에서 보인다. 덕지덕지 4종(3번째 모드·수식·차트·수동 raw/clean)을 명시 제거해 제품이 가볍다. 공개→로컬 funnel이 export tier 차이에 자연 배치된다. 단계가 엔진→공개→터미널로 순차라 각 단계가 독립 출시 가치를 갖는다.
