# 00. 제품 비전 — 기존 터미널에서 *묶는 개념*이 빠졌다

상태: 비전 PRD v0.2
범위: 왜 *분석들을 묶는 수평적 개념*(워치리스트·커맨드·모니터)이 진짜 빠진 것인가, 우리 사용자의 monitor 루프, "공시 워치 ≠ 가격 워치", 왜 지금 개선 ROI 가 높은가. (기존 터미널 개선이지 새 제품 아님.)

---

## 1. 문제 — "깊은 분석 웹앱이지 터미널이 아니다"

dartlab /terminal 은 수직 분석이 *블룸버그 초과 수준*이다: 증권사급 차트(다수 지표·백테스트·비교 오버레이·드로잉·볼륨프로파일·AVWAP), 한국·미국 원문 공시 forensic(IS/CF/BS·배당·자사주·소유구조·인력 정규/계약/성별/근속/1 인당·임원보수·지배구조·**공급망**·R&D), 매크로 쿼드런트·섹터 히트맵·통합 스크리너, 공시뷰어 + grounded Q&A 코파일럿. **블룸버그 `FA`·`GP`·`CN` 을 *각각* 이긴다.**

그런데 사용자 경험은 "터미널"이 아니라 **일회성 조회 웹앱**이다. 이유는 하나다 — 이 깊은 분석들을 *하나의 명령 문법과 모니터 루프로 묶는 수평 층이 비어 있다.* 사용자는 매번 한 회사를 검색해서, 한 보드를 스크롤하고, 떠난다. **다시 돌아올 이유(monitor)가 없다.**

FactSet·Capital IQ 가 분석은 더 깊은 영역도 있으나 "블룸버그 같다"는 평을 못 듣는 이유가 정확히 이것이다: 블룸버그의 정체성은 분석 깊이가 아니라 **함수문법 + 워치리스트 + 워크스페이스 + 모니터의 수평 직조**다.

---

## 2. 우리 사용자 (정확히)

- 한국 리테일 + 세미프로 투자자: 개인 투자자·소형 운용·애널리스트 지망·재무 스터디. 블룸버그 단말($24k/년) 구독 불가. **무료** 도구를 찾는 사람.
- 성향: **매매 타이밍 트레이더가 아니라 펀더멘털·공시 forensic 성향.** 의사결정 주기가 분기 공시·사업보고서지 분 단위 틱이 아니다.
- 우리 데이터 제약: **EOD 일배치(실시간 0), 애널리스트 컨센서스/목표주가 0, 매매체결 0.** 그러나 공시 깊이는 압도적.

이 사용자 정의가 모든 결정을 가른다. *일반 리테일*이었다면 가격 워치·실시간이 1 번이겠지만, *우리 사용자*에겐 공시 델타가 진짜 트리거다(§4).

## 3. 우리 사용자의 monitor 루프 — 어디서 끊기나

블룸버그의 `monitor → research → analyze → execute` 를 우리 사용자용으로 번안한다(execute 없음 → judge → record):

| 단계 | 사용자 행동 | 현재 상태 |
|---|---|---|
| **① WATCH** (지켜본다) | 분석 스터디 중인 10~30 개 회사를 "내 회사들"로 묶어둔다. 재방문 시 *그동안 뭐가 바뀌었나*를 5 초에 본다. | **❌ 치명적 끊김.** 워치리스트 자체가 없다. localStorage 는 차트습관·마지막 종목(`dlTerm.lastSym`)·템플릿·드로잉은 저장하나 *"내 회사들" 집합* 개념이 0. 들어올 때마다 스크리너에서 회사를 다시 찾아야 함. |
| **② SURFACE** (델타가 뜬다) | 지켜보던 회사에 새 공시·재무 업데이트가 *능동 탐색 없이* 뜬다. | **❌ 끊김.** 공시 레일·신선도 측정은 *회사를 이미 연 상태*에서만. "마지막 본 시점 이후 신규"라는 델타 개념이 없다. 어느 게 새 건지 사용자가 직접 기억. (→ 절대시간 신선도는 02 Tier 1[기기독립 근사], 진짜 재방문 델타는 Tier 2[가드 하]가 닫는다.) |
| **③ DIG** (파고든다) | 새 공시를 원문(뷰어)에서 읽고 재무·인력·타법인출자·공급망으로 깊이. 모르면 AskDrawer 에 묻는다. | **✅ 초과 수준.** 손대지 않는다. 유일한 미세 끊김 = forensic 패널로 *바로 점프*하는 길(우측 스택 스크롤로 찾아야 함) → 커맨드 섹션점프가 해소(03). |
| **④ JUDGE** (판단한다) | "이 숫자가 동종 대비 어디?"(compare) · "지금 가격이 이미 가정한 성장률은?"(reverseDCF)로 앵커링을 깬다. | **△ vertical 미배선.** 엔진(`compare`·`reverseImpliedGrowth`) 완성이나 터미널 UI 진입점 약함. **단 이 단계는 financial-statement-lab 소유** — 본 PRD 는 *연결*만(03 §non-encroachment). |
| **⑤ RECORD/SHARE** (기록·공유) | 판단 근거(이 표·차트·공시)를 저장하거나 스터디·SNS 로 내보낸다. | **△ 약함.** 저장 워크스페이스 0. egress = **table-export PRD 소유**. 본 PRD 는 워치리스트가 흡수하는 최소 핵심만. |

**척추 결론: ③DIG 만 완성, ①WATCH·②SURFACE 가 비어 루프가 닫히지 않는다.** 사용자는 매번 ③에서 시작해 ③에서 끝난다 — 회귀 방문의 이유가 0. **본 PRD 의 본질 = 비어 있는 ①WATCH·②SURFACE 를 닫아 터미널의 *재방문 이유*를 만드는 것.** ④JUDGE·⑤RECORD 는 기존 vertical PRD 가 채우며, 본 PRD 는 그것들을 루프에 *연결*만 한다.

## 4. ★공시 워치 ≠ 가격 워치 (정체성 결정)

**우리 사용자의 터미널은 "주가를 지켜보는 단말"이 아니라 "내가 분석한 회사들에 *무엇이 새로 공시됐는지*를 지켜보는 단말"이다.**

블룸버그의 monitor 레이어는 *가격 틱*을 감시한다. 우리는 *공시 델타*를 감시한다 — 그리고 이건 EOD·서버리스 제약 안에서 우리가 블룸버그보다 **더 잘 할 수 있는 유일한 monitor**다:

- **가격 monitor 는 우리가 절대 못 이기는 게임**(실시간 0). 흉내내면 거짓 약속.
- **공시 monitor 는 우리만 데이터가 압도적인 게임**(`RegularFiling`·`NonRegularFiling`·`report` parquet·`syncStatus` lastCommit 전부 라이브). EOD 일배치 = 공시도 일배치라 *제약과 궁합이 완벽*.
- 우리 사용자는 타이밍 트레이더가 아니라 forensic 분석가 — 가격 1% 틱보다 "이 회사에 자사주 소각 공시 떴다 / 사업보고서 갱신돼 인력·배당 숫자 새로 들어왔다"가 진짜 트리거.

**즉 워치리스트와 알림이 우리에겐 *같은 것*이다.** 서버가 없어 푸시는 불가하지만, "내 회사들 + 재방문 시 신규 공시 델타"는 정적 환경에서 100% 가능하고, 이것이 터미널 루프를 닫는 유일한 monitor 다. → [02-watchlist-disclosure-watch.md](02-watchlist-disclosure-watch.md).

## 5. 왜 지금 ROI 가 비정상적으로 높은가

블룸버그의 *묶는 층*이 의미 있는 건 *그 아래 데이터/함수가 충분히 많아서*다(데카르트 곱이 커야 함수문법이 보상받는다). 보통 제품은 이 아래층(콘텐츠)을 쌓느라 수년을 쓴다.

**dartlab 은 운 좋게 그 아래층(수직 분석)을 이미 두껍게 쌓았다.** 즉 우리는 **"비싼 절반(콘텐츠)을 이미 끝내고, 싼 절반(수평 직조)을 안 한" 희귀한 위치**다 — 이 묶는 층 투자의 ROI 가 비정상적으로 높다. 워치리스트 하나가 이미 가진 forensic·차트·compare·reverseDCF 를 전부 "내 회사들"이라는 컨텍스트로 묶어 *루프를 닫는다.* ("싼 절반"은 *공짜*가 아니라 *거의 공짜* — 워치리스트는 신규 포트 메서드 1 개[기존 parquet cross-company 리더] + LeftRail 패널이 전부다. 02 §5.)

단 절제가 핵심이다: 블룸버그의 묶는 층 *전부*를 따라가면(멀티모니터 Launchpad·풀 mnemonic·푸시 알림·트레이딩 데스크) 우리 배포모델(정적·서버리스·EOD)에서 대부분 cargo-cult 가 된다. 따라야 할 선은 블룸버그가 아니라 **Koyfin/TIKR**(현대 무료 리테일 터미널)다 — 큐레이션·섹션점프·범위 표기 델타는 가지고, 트레이딩 데스크·풀 런치패드·실시간 틱은 버린다. → [01-bloomberg-concept-gap.md](01-bloomberg-concept-gap.md) §절제선.

## 6. 한 줄 비전

**워치리스트를 *공시 워치*(재방문 델타, 정적 환경에서 우리만 가능한 monitor)로 정의하고, 커맨드바를 *forensic 자산 주소창*으로 좁혀, 이미 초과 수준인 수직 분석(③DIG)과 기존 vertical PRD(④JUDGE·⑤RECORD)를 하나의 모니터 루프로 직조한다. 이것이 우리 사용자가 블룸버그를 *흉내가 아니라 우회로 이기는* 터미널의 척추다.**
