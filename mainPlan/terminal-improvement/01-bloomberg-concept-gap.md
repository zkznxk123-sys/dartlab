# 01. 블룸버그 개념 갭 분석 — 원시요소 카탈로그

상태: 비전 PRD v0.2
범위: 블룸버그식 터미널의 *개념적 원시요소*(기능 나열 아님) 카탈로그 + 중심성 등급 + dartlab 보유여부 + 현대 리테일 절제선.

## 1. 원시요소 카탈로그 (멘탈 모델 단위)

> 등급: **정의적** = 없으면 터미널이 아님 / **핵심** = 프로 터미널 대부분 보유 / **주변부** = 있으면 좋음.
> dartlab 보유: 있음 / 부분 / 없음.

| # | 원시요소 | 블룸버그 형태 | 등급 | dartlab |
|---|---|---|---|---|
| A | **함수 문법** (Command Mnemonic) — 화면을 메뉴 트리가 아니라 *짧은 함수 코드*로 호출. 보안컨텍스트 × 함수의 데카르트 곱. | `AAPL US Equity <GO>` → `DES`·`GP`·`FA`·`DVD`·`CN`·`RV`. yellow keys 로 자산군 지정. | 정의적 | **부분** — `‹GO›`는 *종목 검색창*일 뿐 함수 grammar 아님. 기능(FA/GP/CN)은 다 있으나 "한 줄에서 함수로 점프"가 없어 마우스로 헤맴. |
| B | **보안컨텍스트 × 함수 직교** — "지금 로드된 종목"과 "지금 보는 함수"가 직교 축. | 종목 바꿔도 함수 유지, 함수 바꿔도 종목 유지. | 정의적 | **부분** — `sym` 상태(컨텍스트)는 있으나 함수 축이 없어 전부 한 보드에 동시표시. |
| C | **워크스페이스/Launchpad** — 패널 N 개를 배치·저장·재호출. 종목 바꾸면 연결 위젯 함께 갱신. | Launchpad, `BLP` 워크스페이스. | 핵심 | **없음** — 보드 1 개 고정. 저장·재배치·멀티뷰 0. |
| D | **모니터 + 알림** — 수동 조회 넘어 *능동 감시*. 워치리스트 + 가격/뉴스/공시 알림. | `WATCH`·`PRTU`·`ALRT`. | 핵심 | **없음** — 워치리스트·알림 0. 스크리너는 "상위 80 자동정렬"로 큐레이션 불가. |
| E | **링크/컬러그룹** — 여러 패널을 색 그룹으로 묶어 한 곳 선택 시 그룹 전체 동기 점프. | 컬러채널 A/B/C. | 핵심 | **없음** — 단일 보드라 링크 개념 자체가 없음. C 의 전제. |
| F | **크로스에셋 정체성** — 주식뿐 아니라 채권·FX·원자재·금리·지수가 *동급 시민*. | yellow keys 가 강제. | 정의적(블룸버그) | **부분** — 좌측 크로스에셋 칩(주식·채권·원자재·금)·매크로 쿼드런트 있음. 단 진입점은 주식. 우리 정체성상 *부분 유지가 정답*. |
| G | **Egress/상호운용** — 터미널은 닫힌 정원이 아님. Excel 함수·데이터 추출. | `BQL`·`=BDP()` Excel add-in. | 핵심 | **부분** — table-export PRD 진행 중. |
| H | **시계열/그래프 일급화** | `GP`·`HP`·`G`. | 핵심 | **있음(초과)** — 차트+지표+드로잉+볼륨프로파일+AVWAP+백테스트. |
| I | **단일화면 재무 디스플레이** | `FA`. | 핵심 | **있음(초과)** — IS/CF/BS·배당·자사주·소유·임원·지배구조·R&D·해석가이드. 한국 공시 특화로 우위. |
| J | **뉴스/공시 레일** | `CN`·`NSE`. | 핵심 | **있음(강화 중)** — 공시뷰어 + 공시 이벤트레일(scenario-simulator). 원문 연결 = 차별 강점. |
| K | **컨센서스/추정** — 애널 목표주가·EPS 추정·등급. | `ANR`·`EEO`·`BEST`. | 핵심(블룸버그) | **없음 · 따라올 수 없음** — 데이터 0. reverseDCF 로 *개념 대체*(추정치 자체는 포기). |
| L | **실시간 피드 / Level 2** | 시장깊이·DMA. | 정의적(트레이딩 데스크) | **없음 · 따라올 수 없음** — EOD 구조상 불가. Koyfin/TIKR 도 버림. |
| M | **메시징/커뮤니티** | `MSG`·IB 챗. | 주변부 | **부분** — giscus 토론이 경량 대응. |

## 2. 핵심 가설 — "묶는 층(수평) vs 분석(수직)" 분리 (강하게 지지)

위 카탈로그를 두 축으로 가르면 패턴이 선명하다:

| | **묶는 층 (수평)** | **분석 (수직)** |
|---|---|---|
| 원시요소 | A 함수문법 · B 컨텍스트×함수 · C 워크스페이스 · D 모니터/알림 · E 링크 · G egress | H 차트 · I 재무 · J 공시 · reverseDCF · compare |
| dartlab | **거의 없음** (A·B 부분 / C·D·E 없음 / G 부분) | **있음~초과** (H·I·J 보유, 일부 블룸버그 초과) |

**결론: 가설이 맞다.** 블룸버그를 블룸버그로 만드는 것은 분석 깊이가 *아니다*. dartlab 은 이미 `FA`·`GP`·`CN` 을 *각각* 가졌지만, 그것들을 *하나의 명령 문법과 모니터 루프로 묶는 수평 층이 없어서* "깊은 분석 웹앱"이지 "터미널"이 아니다.

**단 한 가지 보정**: 묶는 층만으로도 안 된다. 블룸버그의 묶는 층이 의미 있는 건 *그 아래 데이터/함수가 충분히 많아서*다. dartlab 은 운 좋게 그 아래층을 이미 두껍게 쌓았다 — 그래서 이 묶는 층 투자 ROI 가 비정상적으로 높다(00 §5).

## 3. 현대 무료/리테일 터미널 절제선 (Koyfin/TIKR/Finchat)

우리는 $24k/년 블룸버그가 아니라 *무료 도구*다. 따라야 할 선은 블룸버그가 아니라 현대 리테일 터미널이다.

**그들이 블룸버그에서 *의도적으로 버린 것* (= 우리도 버림):**
- 실시간 L2/시장깊이·DMA 주문(L) — 전부 버림.
- 채권/FX/스왑 트레이딩 데스크 함수(F 의 트레이딩 부분) — 버림. 매크로 *맥락*만 유지(Koyfin 의 강점도 정확히 이것).
- IB 메시징/Bloomberg Chat(M) — 버림.
- **고밀도 mnemonic 문법(A) — *부분만 유지*.** Koyfin/TIKR 는 풀 함수코드를 *버리고* 스마트 검색 + 사이드바 네비로 갔다. **풀 mnemonic 암기는 리테일 진입장벽.**
- 자유 타일링 Launchpad(C 극단) — 리테일 터미널은 *고정 위젯 대시보드*를 하지 자유배치를 안 한다.

**그들이 *유지한* 핵심 원시요소 (= 우리가 반드시 가져야 할 선):**
- **워치리스트(D) — 예외 없이 전부 보유.** 리테일 터미널의 1 번 원시요소. Koyfin/TIKR/Finchat 모두 워치리스트가 홈.
- 빠른 종목 전환 + 컨텍스트 유지(B).
- 알림(D) — 단 *우리는 서버가 없어 푸시 불가* → 재방문 델타로 번안(02).
- Excel/CSV export(G) — TIKR·Koyfin 핵심 셀링포인트(우리는 table-export PRD).
- 스크리너 큐레이션(D 인접).

**절제선 요약: dartlab 은 Koyfin/TIKR 라인을 따른다.** 풀 블룸버그 단말의 묶는 층(멀티모니터 Launchpad·풀 mnemonic·링크 컬러채널·푸시 알림·트레이딩 데스크)은 *과잉*, 하지만 **워치리스트 + 섹션점프 + 범위 표기 델타 + (별도 PRD)export** 는 리테일 터미널의 *정의적 최소집합*이고 dartlab 은 이 중 워치리스트·섹션점프·델타를 통째로 비웠다.

## 4. dartlab 에 빠진 진짜 원시요소 (블룸버그-일반 순위)

> 기준: 리테일 터미널 정의적 최소집합 ∩ 우리 제약(무료·EOD·컨센서스 0·매매 0)에서 *실현 가능* ∩ 현재 부재. (우리 *사용자 가치* 기준 재순위는 [00 §3], 본 PRD claim 범위 확정은 [03·04].)

1. **워치리스트** — 1 번 리테일 원시요소이자 모든 후속(델타·섹션점프 컨텍스트)의 전제. → **본 PRD 핵심**(02).
2. **함수 문법 → 섹션 점프(약화)** — 기능은 이미 다 있고 디스패처만 없음. → **본 PRD claim**(03, 약화 형태로만).
3. **저장 워크스페이스/멀티패널** — → **DEFER/KILL**(04. 워치리스트가 최소 핵심 흡수, 자유 Launchpad 는 cargo-cult).
4. **알림** — → **재정의 한정 생존**(02 §비목표. 푸시 KILL, 범위 표기 델타만).
5. **패널 링크** — → **DEFER**(04. 워크스페이스 의존 + marker→row 는 disclosure-rail PRD 소유).
6. **테이블 egress** — → **table-export PRD 소유**(본 PRD 비범위).
7. **reverseDCF 일급화** — → **financial-statement-lab 소유**(03 §non-encroachment. 본 PRD 는 *연결*만).

**명시적 포기선(따라올 수 없는 원시요소):** 실시간 피드/L2/DMA(L), 애널 컨센서스/목표주가 데이터(K, reverseDCF 로 개념 대체하되 추정치 자체 포기), 풀 멀티모니터 Launchpad·IB 메시징·채권/FX 트레이딩(C 극단·M·F 트레이딩).
