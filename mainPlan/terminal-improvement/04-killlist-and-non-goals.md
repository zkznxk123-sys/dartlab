# 04. KILL/DEFER 리스트 + 정체성 가드 비목표

상태: 비전 PRD v0.2
범위: "강함은 깎아서 온다" — Top 7 후보 중 무엇을 죽이고, 무엇을 미루고, 무엇만 남기는가. 근거. 정체성 가드.

> 적대검증(C)이 무자비하게 공격하고 사용자 렌즈(D)가 가치를 변호한 결과의 *합의*. 블룸버그가 가졌다 ≠ 우리가 필요하다.

## 1. 최종 판정표 (Top 7 후보)

| 후보 | 판정 | 근거 (우리 제약 기반) |
|---|---|---|
| **워치리스트 (공시 워치)** | **★ADD (유일 신설)** | 모든 기존 PRD 미claim(grep 0). 정적·서버리스·EOD 를 온전히 견디는 유일 원시요소. 루프를 닫는 단 하나의 레버. → 02. |

## 2. 정체성 가드 — 하지 말아야 할 것 (우리 사용자 기준)

블룸버그를 흉내내다 우리 사용자에게 무가치하거나 *정체성을 파는* 것:

1. **실시간 가격 monitor·틱 알림·인트라데이.** 데이터 0(EOD) → 흉내 = 거짓 약속. 공시 델타에 베팅, 가격은 종속 컬럼만. 인트라데이는 "진짜 부재"(terminal_chart_audit 실측).
2. **컨센서스·목표주가 흉내.** reverseDCF 를 단일 적정주가로 환원하면 정직성 척추(fin-stmt-lab 가드레일) 정면 위반 = *정체성을 파는* 행위. "적정주가 X 원" 절대 금지 — "가격이 요구하는 것의 읽기"로만.
3. **트레이딩 데스크·주문·포지션·손익 추적.** execute 단계 자체가 없음(거래소 아님). 워치리스트는 *관심* 집합이지 *보유* 집합이 아니다. 백테스팅 Strategy Tester(scenario-simulator)는 *전략 탐색*이지 실매매 아님 — 경계 유지.
4. **풀 런치패드·함수문법 백과사전.** 블룸버그 30,000 함수 흉내 금지. 섹션 점프는 forensic 자산 주소창으로만 좁게.
5. **과한 차트 트레이딩 오버레이.** 이미 `charts/` 에 풍부(`extraIndicators`·`drawStore`·`volumeProfile`). 더 쌓지 말 것 — 우리 사용자는 forensic 이지 차티스트가 아님.
6. **서버 의존 기능을 "있는 척".** 크로스기기 동기화·푸시·협업 워크스페이스는 정적 호스팅에서 *불가능*. 흉내내면 깨진 약속. localStorage 한계를 정직하게 노출.

## 3. 숨은 덕지덕지·이중관리 위험 (차단 규칙)

1. **localStorage SSOT 분열** — 키는 `terminal.watchlist` 1 개로 못박는다. 알림 타임스탬프·레이아웃·읽음표시까지 늘면 이중·삼중관리 + 마이그레이션 부채. 나머지 KILL 이 이 분열을 원천차단.
2. **compare verb 충돌** — 멀티심볼 보드 KILL 로 차단. compare()가 N 사 비교 정본.
3. **공시레일 중복** — "신규 공시"는 disclosure-rail 정본, 워치는 카운트만 참조.
4. **fin-stmt-lab reverseDCF 중복** — 소유 고정, 터미널은 표시만.
5. **"배선만 하면 됨" 과소평가 함정** — 포트는 *계약 실재 but 미배선 + 터미널 미소비*(03 §2). 비용을 "싸다"로 통과시키지 말 것. 워치리스트 외 후보는 비용 재평가 시 ROI 붕괴.

---

## 4. 한 줄 종합

강함은 7 을 1 로 깎아서 온다. 워치리스트(공시 워치) 하나만 신설하고, 그것이 섹션점프·정직 델타를 흡수한다. 멀티패널 Launchpad·푸시 알림·mnemonic 백과사전·크로스에셋 instrument 타입·reverseDCF 함수승격은 우리 배포모델(퍼블릭 정적·서버리스·EOD)과 기존 PRD 소유권 앞에서 전부 KILL/DEFER다. 블룸버그식 *묶는 층* 욕구의 대부분은 우리 환경에서 cargo-cult이며, 정직성 가드가 생존선이다.
