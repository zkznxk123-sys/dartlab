# 00. Product PRD — 정직 백테스트 OS

상태: v0.2 (2026-06-18 — 8 전문에이전트 2라운드 토론 수렴)

---

## 1. 제품 정의

**Terminal Strategy Lab = 한국 DART panel을 두 방향으로 자르는 세계 최고 수준의 *정직한* 무료 EOD 백테스트 OS.** 두 입구 — ① 단일종목 시간기계(주가차트 위 다전략·펀더게이트·리플레이·미래연속) ② 유니버스 크로스섹셔널 랭킹(17년 가격보존 전종목). 둘을 묶는 건 화면 통합이 아니라 *동일한 PIT 정직 공리*이며, 단방향 drill-down(유니버스 Q5→단일종목)이 워크플로 깔때기를 만든다.

핵심 차별 = **능력이 아니라 정직.** TradingView·QuantConnect·Bloomberg는 능력은 위지만 정직은 toy다 — 생존편향 처리가 불투명하고, 단일 path Sharpe를 *유의*인 양 헤드라인하고, in-sample 미인대회를 "전략 발견"으로 포장하고, folk-stat 천장(DSR/PBO)을 *팔 이유가 없어* 안 보여준다. 본 OS는 그들이 *상업적으로 숨기는 것*(생존편향 밴드·selection 카운터·OOS 강제 비활성불가·folk-stat 봉인)을 1급 표면으로 박는다. **그 자리는 비어 있다 — 능력으로 세계급을 노리면 toy 폭로로 끝나고, 정직으로 노리면 도달 가능.**

부차 차별 = **"숫자 리포트"가 아니라 "시간/횡단면 위의 시각적 증명"**(성과가 차트와 같은 좌표 + 항상 거기 + 가정 안 숨김). 단 이건 *UX 주장*이지 데이터 moat 주장이 아니다(TradingView도 차트는 강함). 진짜 데이터 moat = panel(펀더게이트·17년 가격보존 유니버스).

## 2. 사용자 · 사용 맥락 (교집합 — terminal-improvement 충돌 해소)

> **타깃 = "무료 도구를 쓰는 한국 세미프로 — 펀더멘털로 회사를 판단하되, 그 판단이 *시간과 횡단면 위에서 실제로 작동했는지*를 규칙으로 검증하려는 사람."**

- **1차(forensic + 검증 모드)**: 재무로 회사를 판단하는 사용자가 그 판단을 백테스트로 검증. "Z-score 위험구간 회사는 가격도 무너졌나?", "Piotroski≥6일 때만 골든크로스로 사면?"을 17년 데이터로. **펀더게이트(§4.10)가 이 모드의 다리.** `terminal-improvement` 사용자 정의(분기 공시 forensic·타이밍 트레이더 *아님*)와 일관.
- **2차(순수 기술 트레이더)**: 차트만, 재무 무관. commodity 캔버스만 제공, moat는 1차 겨냥. dartlab은 EOD·실시간0이라 이 세그먼트엔 구조적으로 빈약한 토양 — 끌어오려 하지 않는다.
- **로컬 리서치**: walk-forward·민감도·MC로 과적합 진단(Python bonus).
- 사용 맥락: 터미널 메인 주가차트(005930 등) = 단일종목 입구 / scan 유니버스 백테스터 = 횡단면 입구.

## 3. 목표 종착 (이게 전체 그림 — 단계는 여기로 진입)

1. **다전략 캔버스**: N전략(≤3)이 같은 차트에 색별 진입/청산 마커 + 공유 절대축 에쿼티 곡선으로 공존. 동일가중 조합 곡선이 개별보다 위/아래에 있는지, 조합 MDD가 개별보다 얕은지(분산효과)가 눈에 보임.
2. **전문 거래 분석**: 손절/익절/트레일링·포지션 사이징, MAE/MFE·expectancy·R-multiple·연속승패·롤링 Sharpe·월별 수익 히트맵·낙폭/회복 분석.
3. **시간 walk + 리밸런싱**: 리플레이로 과거를 재생하며 전략 결정을 실시간 관찰. 재생 중 *그 시점까지만 보이는* 상태에서 가중치를 바꿔(커서 바인딩) 재량 운용을 정직하게 walk.
4. **강건성 진단(로컬)**: walk-forward(재학습)·파라미터 민감도 히트맵·Monte Carlo 거래 재배열·CPCV로 "이 성과가 운인가 패턴인가"를 진단.
5. **미래 연속**: asOf의 포트 상태(전략별 포지션·자본·가중치)가 시뮬 Play fan의 초기조건이 되어, 과거 검증 곡선이 미래 시나리오로 끊김 없이 이어짐.

## 4. ★전문 백테스팅이란 무엇인가 — canon 개념 카탈로그 (운영자 말을 넘어서)

전문 백테스팅 정전(Pardo, Bailey/López de Prado, Chan)의 개념을 전수 검토해, *우리 단일종목·브라우저·정직 가드*로 거른다. **adopt / 조건부 / reject(folk-stat·단일종목 무의미)** 3분류.

### 4.1 체결 현실성 (Execution realism) — adopt 강화
| 개념 | 판정 | 비고 |
|---|---|---|
| t종가 신호 → t+1 시가 체결 | **adopt(기존)** | look-ahead 구조 차단. 불변. |
| 수수료·거래세·슬리피지 bp | **adopt(기존)** | 한국 기본값. 편집 가능. |
| 거래정지(v=0) 체결 이연 | **adopt(기존)** | 감사 카운터 노출. |
| 인트라바 손절 체결(당일 저가 터치) | **adopt(P2)** | ⚠ "당일 인트라바 가정" 라벨 강제(t+1 체결 모델과 시점 충돌 명시). |
| 거래량 참여 한도(ADV %) | **조건부(P4 로컬)** | 단일종목 소액엔 과함. 로컬 정밀 모드 옵션. |

### 4.2 포지션·리스크 관리 — adopt
| 개념 | 판정 |
|---|---|
| 손절/익절/트레일링 스탑 | **adopt(P2)** — long/flat 유지, exitReason='stop'. |
| 시간 기반 청산(N봉 후) | **adopt(P2)** |
| 포지션 사이징 — fixed-fractional, vol-target | **조건부(P2)** — *규칙적 리스케일*은 adopt. |
| Kelly fraction | **reject 수치** — 표본 모멘트 추정 분산 폭발(단일종목 folk-stat). 개념 설명만, 수치 제시 금지. |

### 4.3 거래 분석 (Trade analytics) — adopt
MAE/MFE(최대역행/최대순행)·expectancy(기대값)·R-multiple·win/loss streak·보유기간 분포·거래수익 히스토그램. **전부 adopt(P2)** — 거래 단위 사실, 통계 추정 아님. expectancy·R-multiple은 전문 트레이더 핵심어인데 현 제품에 없음(운영자 말 넘어선 추가).

### 4.4 위험/수익 분석 — adopt
Sharpe/Sortino(기존)·Calmar·MAR·Ulcer Index·tail ratio·롤링 지표(63봉)·underwater 곡선·낙폭/회복 분석. **adopt(P2)** — 단, 표본<60봉 null 게이트 상속. Calmar/Ulcer는 낙폭 민감 지표(전문 추가).

### 4.5 벤치마크 상대 — adopt
alpha/beta/IR(기존)·up/down capture ratio·tracking error·active return. **adopt** — 동일비용 B&H 대비 의무.

### 4.6 강건성/검증 — 조건부·로컬 (folk-stat 천장)
| 개념 | 판정 |
|---|---|
| IS/OOS 분할(고정 파라미터) | **adopt(P1, floor)** — 표본이 받침. test<train 경고. |
| Walk-forward(재학습) | **조건부(P4 로컬)** — 단일종목 폴드 거래 0~2건이면 무의미 → 폴드별 거래수 동반. |
| 파라미터 민감도 히트맵 | **adopt(P4)** — ⚠ **argmax/"최적값 추천" 금지**. "어떤 값도 위험"의 과적합 교구. |
| Monte Carlo 거래 재배열/부트스트랩 | **adopt(P4)** — 경로의존성 스트레스. "거래 순서가 운이었나"의 정직한 시각화(전문 추가, 운영자 말 넘어섬). |
| CPCV(purged) | **조건부(P4 로컬)** |
| DSR / PBO / Deflated Sharpe / haircut | **reject 수치(단일종목)** — nTrials≈6·2020+ 표본에서 노이즈(folk-stat 천장 §0.6.1). 다전략·로컬 게이트 통과 후만. |

### 4.7 다전략/포트폴리오 — 조건부 (단일종목 한계)
| 개념 | 판정 |
|---|---|
| 전략 조합(자본 배분 슬리브) | **adopt(P1)** — equity 가중합. |
| 동일가중 | **adopt(P1)** · 역변동성 = **조건부(P3, lookback selection 자유도)**. |
| 리밸런싱 | **adopt(P3)** — 커서 바인딩 필수(04). |
| 종목간 분산효과·상관·위험패리티 | **reject(단일종목)** — 분산효과 0. *다종목*에서만 정직(별도 후속). 단일종목 조합 = "타이밍 분산이지 자산 분산 아님" 라벨. |

### 4.8 미래/시나리오 — 연결만
Monte Carlo 전방 투영·스트레스 시나리오 = `scenario-simulator/` 소유. 본 랩은 asOf 포트 상태를 *초기조건으로 넘기는* 연결(P5)만.

### 4.9 ★프리셋 다양화 — close-only 6개 너머 (실측 기반 재사용)
현 6 프리셋(maCross·rsiRevert·bbRevert·macdCross·donchian·momentum)의 약점 = **전부 종가(close)만 쓴다**(거래량·고저 무시 → fake breakout·고정폭 손절 취약). `synth/indicators`(vatr·vobv·vstochastic 등 38+)·`quant/signal`(volume/momentum/event/flow)·`quant/strategy/styles`(8종) **재사용**으로 다양화:
| 후보 | 차별 | 입력 | 단계 |
|---|---|---|---|
| 거래량 확인 MA(VWMA+OBV) | fake breakout 제거 | close+vol | P1.5 |
| 변동성 필터(모든 진입 게이팅) | 추세시작 spike 회피 | realized vol | P1.5 |
| ATR 동적 손절(전 프리셋 옵션) | 고정폭 탈피 | high/low | P2 |
| Supertrend · Stochastic RSI · ROC | 지표 다양화 | OHLC | P1.5 |
| 스타일 8종(trendFollow·meanReversion·breakout 등) | 학술 근거 | quant/styles 재사용 | P2 |

### 4.10 ★단일종목 펀더 게이트 — moat (실측 확정, 차선의 한방)
유니버스 팩터 *랭킹*은 생존편향(§아래)으로 막혔으나, **한 종목의 재무 건강이 좋을 때만 진입**은 단일종목이라 생존편향 무관 + PIT 근사만 하면 정직. 가격 백테스터가 못 하는 panel moat.
- 예: "Piotroski≥6 **이고** 골든크로스일 때만 매수" · "Altman Z 위험구간이면 진입 금지" · "Beneish 이익조작 의심이면 청산".
- 재사용: `quant/alphas/` **학술 팩터 9개 이미 구현**(Piotroski F·Altman Z·Beneish M·QMJ·BAB·q-factor·earnings surprise·fundamental momentum·accruals).
- 시각화 = 가격 페인 **배경 음영**(02 §1.5 ①, 분기 계단·PIT 근사). 백테스트 진입 *게이트*(조건 빌더 항)로 결합 — 단일종목 랭킹 아님.

## 5. 성공 지표 (차트 수·KPI 수 아님)

- **정직이 권위로 읽히는가**: 정직 라벨이 *차갑게 이탈을 유발*하지 않고 *신뢰·전문성*으로 읽히는가(02 §4 — Bloomberg/Koyfin 면책 패턴). 정직 문구가 가장 안 읽히는 9.5px가 아니라 *그것이 한정하는 숫자만큼 legible*(11px floor).
- **눈으로 비교 가능한가**: 두 전략 + 조합이 *공유 절대축*에 "+30% vs +12% vs 조합 +22%"가 한 자에서(per-series 정규화 함정 회피).
- **유니버스 분위가 novice-legible한가**: "분위가 단조롭게 벌어지나" + "내 종목이 분포 어디"가 한 자에. 폐지 양극단 밴드로 *불확실성까지* 정직.
- **never-claim 0건**: 0.1 §0.1 / 04 §6 grep 게이트 — "세계급/기관급"·"survivorship-clean" 무수식·단일종목 Sharpe를 "전략 성과"로·유니버스 t-stat 수치·"검증된 팩터"·펀더게이트 "moat" 0건.
