# 02. UI/UX Design — 세계급 재설계 (persistent dock + progressive disclosure)

상태: v0.2 (2026-06-18 — UX 전문에이전트 재설계). 거처 = `ui/packages/surfaces/src/terminal/` + `ui/packages/surfaces/src/scan/universe/`.

> 원칙: 차트가 1차 표면. 컨트롤 = **차트 레이아웃의 일부인 persistent dock**(자동닫힘 금지). 정보는 **점진적 공개**(Glance→Compose→Decompose→Diagnose)로 한 번에 하나씩. 정직 라벨 = 권위(11px floor·3단 tiering). 가독 하한 10.5px.

---

## 0. ★현 실태 진단 (코드 grounding — v0.1 설계가 틀린 전제 위에 섰음)

v0.1 02는 `StrategyConsole` 인셋이 있고 `BtConfig`가 폐기됐다 가정했으나, **코드는 반대다 — 실태가 더 나쁘다**:

1. **config가 *자동 닫히는 드롭다운*** — [ChartMenus.svelte:214-217](../../ui/packages/surfaces/src/terminal/charts/ChartMenus.svelte) `[BT]` 버튼이 `menu==='bt'` 플라이아웃에 `BtConfig` 전체(룰빌더+비용+OOS)를 마운트. 차트를 클릭해 결과를 보는 순간 빌더가 *사라진다*. **tweak→see 핵심 루프가 구조적으로 깨짐 — 단일 최대 실패.**
2. **moat 기능이 invisible** — [BtConfig.svelte](../../ui/packages/surfaces/src/terminal/charts/BtConfig.svelte)는 강력한 조건빌더(지표·연산자·crossUp/Down·AND/OR·조건 추가/삭제, `RULE_PRESETS`·`SERIES_CATALOG` 배선)인데, 출력이 *차트 어디에도 없다*. `BT_TRADES`는 진입/청산 삼각형만 → "왜 여기선 안 샀지"(어느 조건이 꺼졌나)가 관측 불가. 빌더 가치가 렌더 시점에 버려짐.
3. **3개 경쟁 결과 표면, 권위자 없음** — `BacktestStrip`(1줄 헤드라인) + `BacktestDialog`(모달) + 인차트 `BT_EQUITY` 범례. 같은 숫자가 3곳에 3가지 크기로. 점진 공개가 아니라 *병렬 공개*.
4. **정직 문구가 9.5px** — `BacktestStrip.btSel/.btFoot/.btWarn`이 정직 텍스트를 9.5px(02 v0.1 §6 자기 하한 10.5px *위반*)로 렌더 → **권위로 읽혀야 할 정직이 화면에서 가장 안 읽힘. 의도 역전.**
5. **진입이 파괴적 side-effect** — `[BT]`가 조용히 `tf='D'`·`period` 강제 전환. 2자 버튼 뒤 빈 empty-state.

→ v0.2 = 이 5개 실패를 *기반 단계*(persistent dock + 정직 tiering)에서 먼저 닫는다. 이게 전 탭(단일·유니버스·게이트)의 토대.

## 1. ★진입 모델 — 하나의 persistent dock, 세 모드

드롭다운을 죽인다. **차트 레이아웃의 일부인 docked 좌측 레일**(리사이즈·접힘 28px spine, *자동닫힘 0*)로 교체. TradingView Strategy Tester의 통찰 = config·result·chart가 *동시에·영구히* 보여야 한다. dock은 한 시각 언어를 공유하는 입구를 가진다:

| 모드 | 역할 | 거처 |
|---|---|---|
| **① 단일종목 Lab** (기본) | 현 종목, N≤3 전략, 룰빌더, 인라인 결과 | `terminal/charts/` (StrategyDock) |
| **② 유니버스** | 포커스 전략을 17년 가격보존 횡단면에 실행 | `scan/universe/` (별도 폴더, 단방향 drill-back) |
| **③ 게이트** (탭 아님, *수식자*) | 펀더 게이트, 전략별 토글, 가격페인 배경 음영 | 룰빌더 조건 종류 |

한 색 시스템(`STRAT_COLORS` + combo amber + B&H 회색 점선), 한 헤드라인 문법(`X% vs B&H Y% · +Zp · ⚠label`), 한 deep-link 제스처(무엇이든 클릭→그 봉/기간으로 차트 스크롤).

## 2. ★점진적 공개 사다리 (핵심 원칙)

> **Glance → Compose → Decompose → Diagnose.** 각 레벨 opt-in. 기본 = Glance.

| 레벨 | 표시 | 트리거 |
|---|---|---|
| **Glance** | 가격 + 포커스 거래 마커 + 에쿼티 서브페인 + 1줄 헤드라인 | dock 열기(기본) |
| **Compose** | + dock 룰빌더 행 + 펀더게이트 음영(게이트 항 존재 시) | 전략 추가/편집 |
| **Decompose** | + 포커스 전략의 **조건 레인**(opt-in, 조건의 ◉ "차트에 표시" 클릭 시 가격 밑에 lane 드롭) | 조건 ◉ 핀 |
| **Diagnose** | 전체 다이얼로그: MAE/MFE·expectancy·R-multiple·OOS·민감도(로컬) | `[상세 ▸]` |

02 v0.1 §1.5의 레인 스택은 **전부 Decompose에**, 그것도 *사용자가 명시 핀한 조건만*. 기본 = 가격+에쿼티+낙폭(3밴드, 오늘과 동일). 이게 인지 과부하 반론을 해소하면서 "아하"를 보존.

## 3.1 ★ASCII 목업 ② — Decompose (조건 레인, opt-in)

조건마다 `◉`를 클릭하면 가격 밑에 *핀한 조건만* 얇은 레인이 드롭. 02 v0.1 §1.5의 정렬 레인을 점진화:
```
┌─ 가격 페인 ──────────────────────────────────────────────────────────────┐
│ ░░░ Piotroski≥6 (게이트 음영) ░░░   ▲진입      ✖손절   ▼청산                │
│   캔들 + MA200 (조건에 쓰인 지표만 오버레이)                              │
├─ RSI(14)<30      ▮▮░░░░▮▮░░░▮▮░░░░  ◉ ──────────────────────[숨김]──────┤ ← 핀
├─ 가격 cross↑ MA200 ░░░▮▮▮▮▮▮▮▮▮░░░  ◉                                    ┤   레인
├─ = 진입(AND)      ░░░░░░▮▮░░░▮░░░░  ══→ ①의 ▲진입과 수직 정렬           ┤ ← 합성
├─ 에쿼티 (공유축)  조합━ A── B··· B&H- -                                  ┤
├─ 낙폭 ▔▔╲▁▁╱▔▔                                                          ┤
└──────────────────────────────────────────────────────────────────────────┘
     리플레이 커서 t ──┃  (전 레인 동시 절단·동기 walk — look-ahead 상속)
```
"아하" = *"왜 여기선 안 샀지?"* → `RSI<30` 레인이 어두웠음(꺼짐)이 보인다. **규칙의 실패가 시간 위에서 가시.** Pine/C#/블록은 코드 뒤에 숨김. LOD: 줌아웃→합성(AND)만, 줌인→핀 조건 펼침. 비핀 조건은 안 그림(클러터 floor).

## 4. ★정직 가드를 권위로 (차가운 면책 → 전문성)

20+ 경고는 dartlab의 *TradingView 카지노 느낌에 대한 moat* — 단 전문성으로 읽힐 때만. Bloomberg/Koyfin 패턴:

1. **흩어진 20개 칩 → 하나의 persistent 정직 줄**. 오늘 `.btSel`·`.btFoot`·`.btWarn`·`bdBanner`·`bdAssume`·`bdFoot` 6표면 9.5px → 헤드라인 밑 *항상 보이는 한 줄*(11px, 중립 slate)이 *spec stamp*로: `t+1 시가체결 · 비용반영 · 배당미반영 · 단일구간`. amber는 *활성 문제*에만.
2. **가정을 feature로 승격, 경고에서 강등**. Bloomberg PORT처럼 방법론을 권위로. "⚠ next-open fills"(무섭게) 아닌 **"체결 모델: t+1 시가 (미래참조 차단)"**(조용한 역량 신호). `ⓘ 방법론` 칩이 전체 가정 원장(`bdAssume`)을 on-demand.
3. **3단 severity, 시각 구분**: *spec*(중립, 항상 — 체결모델) / *caution*(slate italic — 단일구간·배당) / *active warning*(amber, 트리거 시만 — 거래<10·split의심·test<train). 오늘은 전부 amber blur → tiering = 사용자가 중립 spec을 *신뢰*하고 amber에 *주목*하도록 학습.
4. **추론 지점에 inline, footer 덤프 아님**. OOS test<train은 *곡선 분할선 위*(btLayer 이미 반쯤, "학습/검증" 라벨), "타이밍 분산" 라벨은 *combo 곡선 옆*. 맥락 anchored 정직 = 통찰로 읽힘 / footer 정직 = 법률문구.
5. **정직이 legibility floor 밑으로 절대 안 감**. 9.5px 정직 텍스트 = "건너뛰길 바라는 fine print" 신호 = 권위의 정반대. 세계급 = 면책이 *그것이 한정하는 숫자만큼 legible*. **11px로 상향**.

## 5. P3/P5 — 리밸런싱·미래연속 (이연 — 설계만)

- **P3 리밸런싱(D1)**: dock 가중치 바, **리플레이 일시정지 시만 활성**(재생 중·전체차트 잠금 + "되감아 그 시점에서만"). ⚖ 타임라인 마커·append-only(되감아 수정=재포크)·반복재생 카운터.
- **P5 미래연속(D3)**: 리플레이 끝봉 → `mode:simulate`. asOf 우측 fan band, 조합 곡선 끊김 없이. 미래=점선·반투명·"가정 구간" 워터마크·단일 path 금지(밴드). path 데이터 = 시뮬 코어 졸업 후.

## 6. 타이포·색·인터랙션 규칙

| 역할 | 크기 |
|---|---|
| L1 헤드라인 값 | 16–17px bold mono |
| L2 KPI 카드값 | 16–17px · 칩/표 본문 11.5–12px |
| L3 라벨·섹션헤더·**정직 각주** | **11px(상향 — 9.5px 위반 전수 교정)** · 레인 라벨 10.5px(하한 허용) |
| 금지 | **<10.5px**(값·라벨·정직 — 10.5px 미만). SVG 축 눈금만 예외 |

- 색: `STRAT_COLORS`(≤3, 녹/보라/cyan) — **전략 ID 해시 고정(성과 무관)** + 조합 amber 굵게 + B&H 회색 점선 + 손절 적색 ✖. **승자 강조 금지**(굵기·정렬·색에 우열 신호 0, 정렬=입력순 고정). 04 §6 grep "수익순 정렬" 회귀.
- 인터랙션: dock ◆=포커스 / ⚙=스탑·게이트 / ✕=삭제 / ＋=추가(≤3) / ◉=조건 차트핀. 거래행 클릭→차트 봉 센터링(리플레이 유지). dock 접힘=28px spine(헤드라인만).
- 정직 표면 상존(닫기불가, §4 tiering): spec 줄 · selection 경고 · 단일종목 분산 라벨 · 표본<10 빨강 · 배당미반영 · "과거 가정 노출형 시뮬레이션·추천 아님" · "세계급/기관급" 문구 0(grep).
