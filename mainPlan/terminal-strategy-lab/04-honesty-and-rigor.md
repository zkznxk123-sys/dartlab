# 04. Honesty & Rigor — 통계 정직 SSOT

상태: v0.1 (2026-06-16). 옛 `_done/terminal-chart-suite/03 §0.6`을 승계·확장. 본 문서 = 백테스팅 정직 가드 정본(시뮬이 재사용).

> 한 줄: **"강해 보이는 화면"이 "단일구간 in-sample 결과 과신"으로 이어지는 것을 막는다. 화려함이 아니라 가정을 숨기지 않음이 권위다.**

## 1. 승계 가드 (옛 §0.6 — 불변)

- **G1 look-ahead 구조 차단**: 신호 t일 종가 → t+1일 시가 체결. displaySeries 절단이 N전략·리밸런싱·미래에 *구조적 상속*.
- **G2 통계 노출 천장(folk-stat)**: 단일종목·6프리셋·2020+ 표본에 **DSR/PBO/CPCV/haircut Sharpe·Kelly fraction 수치 노출 금지**. 코드가 숫자를 뱉어도 유효하다는 뜻 아님(nTrials≈6·폴드 거래 0~2건 = 노이즈). parity gate + nTrials 충분 + 다전략/로컬 동시 충족 전 영구 DEFER.
- **G3 "전문가급"·"추천"·"세계급/기관급" 문구 봉인**: UI·툴팁·코드네임·문서 어디에도. **"세계급/기관급"은 G3의 우회 동의어 — 동일 금지**(능력 주장이 거짓이 되는 선, §6 grep). 사용자 표면 = "**과거 일별 데이터 기반 가정 노출형 시뮬레이션**". 자기규정 = "세계 최고 수준의 *정직한* 무료 EOD 백테스트 OS"(형용사를 능력이 아니라 *정직*에).
- **G4 탐색 = 과적합 경고**: 파라미터/스탑/가중치 슬라이더는 "최고값 찾기"가 아니라 "민감도·과최적화 위험". `best`/`optimal`/`추천` 단어 금지. 민감도 히트맵 **argmax/추천 화살표 금지** + **색칠은 수익이 아니라 *셀 간 분산*으로**("값이 흔들린다=위험"을 보이게, "어디가 높다"를 눈이 argmax 못 하게). 곡선 색=전략 ID 해시 고정(성과 무관), 정렬=입력순(수익순 금지).
- **G5 벤치마크 편향 상시 고지**: 배당 미반영(`dividend:'excluded'`) + 무수정주가 기본이 B&H를 깎아 전략이 상대적으로 좋아 보임 — 일회 아닌 상존.
- **G6 엔진 개선 게이트**: 신규 *프리셋/지표*는 G1-held-out(미래+미학습회사 split)·G2-zero-train(동일비용 B&H 강baseline) 통과 후. factor-zoo 차단.

## 2. 신규 가드 — 다전략·조합·리밸런싱 (본 PRD 추가)

### 2.1 ★리밸런싱 정직 4조건 (P3, 하나라도 빠지면 백테스트 무효)
"결과를 *보면서* 그때그때 리밸런싱" = 손-look-ahead = 무효. 정직화 유일 경로:
1. **커서 바인딩(hard)**: 가중치/결정 입력은 리플레이 재생 중 *t까지만 보이는*(displaySeries 절단) 상태에서만. 전체차트 가시·simulate 모드엔 잠금 + "되감아 그 시점에서만" 라벨.
2. **append-only(hard)**: 되감아 *이미 내린 결정 수정* 금지 → 그 시점 이후 결정·결과 폐기·재포크(새 runId). 되감아 답 보고 고치기 = 손-look-ahead의 본체, 코드로 차단.
3. **반복재생 카운터**: 같은 구간 N회 재생 튜닝 = in-sample 과적합 → 막을 수 없으니 *세어서 노출*("이 구간 N회 보고 조정 = 과적합 위험"). 침묵 금지.
4. **OOS 가림(선택 hard)**: 최상위 정직 모드 = 커서가 train 끝에서 멈추고 test 구간은 입력 잠금+시각 가림. "검증 구간은 손대기 전엔 안 보인다"가 유일한 정직 walk-forward 흉내.

### 2.2 다전략 비교 = selection bias (P1)
N전략을 한 화면에 띄워 "제일 위 곡선 선택" = 사후선택 편향. 정직 조건(전부 필수): **승자 강조 금지**(색·굵기·정렬 중립) · 곡선마다 동일비용 B&H 대비 · OOS 2열 동반 · **selection 경고 상존**("N전략 같은 데이터 비교 = 사후선택 편향. 위 곡선이 미래 최고 아님"). 하나라도 빠지면 cherry-pick 기계.

### 2.3 조합 가중치 (P1·P3)
- P1 = **동일가중 1종만**(데이터 보기 전 결정된 규칙). 역변동성도 lookback selection 자유도 → P3.
- **자유 가중 슬라이더는 과거 곡선 보고 정하면 곡선맞춤** → P3에서만(커서 바인딩 하에 재량 walk). 전체차트 보며 자유조정 금지.
- combo 라벨 = "고정가중 보유합성(리밸런싱 없음)". combo 거래 KPI(승률·PF·노출·거래표) = **명시적 "—"**(억지 합산 = 거짓정보).

### 2.4 단일종목 분산 라벨 (P1)
같은 종목 두 전략 = **타이밍 분산이지 자산 분산 아님**(분산효과 0, 같은 가격 path). "포트폴리오 조합 검증"·"상관"·"위험패리티" 어휘는 단일종목에서 금지 → *다종목*(별도 후속)에서만. 조합 MDD 감소는 "타이밍 차이"로만 라벨.

### 2.5 미래 연속 과신 차단 (P5)
과거 구간 튜닝 전략을 Play 초기조건으로 넘길 때: **튜닝 출처+replayCutDate 운반**("이 전략은 과거 [기간] 튜닝 — 미래 보장 아님 · in-sample 적합" 워터마크). 결정론 path ≠ 검증(전략 실선 과거 / 미래 점선 가정구간 시각 분리). 단일 path 강조 금지(밴드/다분기). 표본·OOS·편향 경고가 mode 전환에서 *증발 금지*.

### 2.6 ★펀더 게이트 PIT + 생존편향 (실측 2026-06-16 — gov date 샤드 직독)
- **가격 생존편향 = 이미 해결됨(2010~2026)**: `gov/prices/date/{year}.parquet`가 "그날 거래된 전종목"을 보존 upsert → 상폐/합병/SPAC해산 회사도 폐지일까지 봉이 남음. 실측: 2010 상장 2073종목 중 618(29.8%)이 2026까지 사라짐, 전부 데이터에 보존. **17년 survivorship-clean 유니버스가 이미 HF에 있어 가격/기술 유니버스 백테스트는 지금 정직하게 가능.** 재수집·재구축 불필요. 잔여 라벨 = 폐지 사유 미구분(상폐/합병/코드변경 혼재, 가격 백테스트엔 무관).
- **재무(panel) 생존편향 = 잔존(실측 13.9%)**: panel 2930개 중 현재 거래 코드는 95.5% 보유하나 사라진 코드는 **13.9%(86/618)만 보유** → 재무 팩터 *랭킹* 유니버스는 망한 회사 재무 결측으로 생존편향 잔존(가격과 달리 clean 아님). 따라서 재무 팩터 유니버스 = 현재유니버스+생존편향 라벨 강제 또는 단일종목 펀더 게이트(clean)까지만. 정밀 재무 유니버스(상폐사 재무 재수집)는 별도 트랙. 단일종목 펀더 게이트는 생존편향 무관.
- **단일종목 펀더 게이트 = 정직 가능**: 한 종목 게이트(Piotroski≥6일 때만 진입)는 *그 종목만* 보므로 생존편향 무관. 단 **PIT 근사 필수** — 펀더 값은 `rcept_dt`(공시일) *이후* 봉부터 적용(계단). 공시 전 봉에 미래 재무를 쓰면 look-ahead. 정정공시 버전 미보존 → "공시일 기준 근사(정정 반영 최신값)" 라벨.
- 펀더 게이트 시각화(배경 음영)는 분기 계단·공시일 시작 — 매끈한 연속선 금지(데이터 띄엄띄엄 사실 노출).

### 2.7 조건 빌더 과적합 (P1.5)
다중 조건 자유 조합 = in-sample 자유도 폭발(단일종목 거래<10에서 조건 4개 AND = 데이터 암기). 가드: **동시 조건 ≤2~3 권장 상한**·OOS 2열 강제(끌 수 없음)·거래<10이면 Sharpe/winRate 수치 숨김·`best`/`optimal`/`추천` 단어 금지·조건 추가할수록 거래수↓ 경고. 조건 빌더는 "최적 규칙 발견"이 아니라 "규칙이 시간 위에서 어떻게 작동하는지 분해"로 프레이밍.

### 2.8 ★유니버스 크로스섹셔널 백테스트 (별도 객체 — 단일종목 가드 비상속)
전종목 랭킹 유니버스(`05-universe-backtest.md`)는 ①(N≤3 손선택 equity합)과 **다른 엔진**이다. §2.1~2.7 가드는 유니버스 특유 거짓말을 못 막으므로(거짓 안전감) 신규 hard 가드 — 상세·근거는 05 §7:
- **U-G1 생존 = 합병식별 + 양극단 이중실행 밴드(hard)**: 폐지 청산가는 사유 미구분이라 알 수 없음 → **임의 숫자(−30% 등) 금지(folk-stat)**. ★**그러나 "사유 미구분"을 미덕으로 두고 출시하면 밴드가 무너진다**(적대검증): 현 `buildUniversePanel.py:128` `delisted`가 합병/정지/코드변경/실폐지를 한 bool로 뭉갬 → **합병(주주 인수가+프리미엄 수령)을 −100%에 넣으면 보수 헤드라인이 *허구로 과소평가*** → 밴드가 "불확실성 노출"이 아니라 "분류버그 증폭기". **수정(U1 출시 필요조건)**: ① allFilings 합병공시로 **합병을 식별 → last-close 처리(밴드에서 제외)**, ② unknown 폐지만 양극단(0손실/−100%) 2회 실행 → **밴드 폭 = *진짜 unknown 폐지 의존도***, ③ 상한은 last-close(0손실)에 고정 + "합병 프리미엄은 이 위 — 낙관 끝도 보수적일 수 있음" 라벨(*비대칭 무지의 방향 정직*: 밴드 전체가 진실의 하방일 수 있음), ④ 밴드 폭>30%p면 hero 숫자 차단(G-M4를 라벨→*차단*으로 격상). ★00 §4.11 "사유 미구분=가격 백테스트 무관"은 단일종목 B&H엔 맞으나 *크로스섹셔널 랭킹엔 거짓*(망할 회사 보유 팩터 과대평가).
- **U-G2 PIT 멤버십(hard)**: 유니버스·필터(시총/거래대금 컷)는 그 리밸 시점 그날 데이터로만(assert ts≤rebalanceT). forward-fill 금지.
- **U-G3 턴오버·ADV P1 승격(hard)**: 턴오버·ADV 초과 주문비율 KPI 동급 상시. 주문>ADV X% = "실거래 불가능" 빨강. 동일가중=소형주 틸트→고턴오버 라벨.
- **U-G4 이중 벤치(hard)**: 동일가중 유니버스 + 시총가중 지수 둘 다 표시 의무. size 틸트 라벨.
- **U-G7 selection 차단**: argmax 금지만으론 부족 → 시도 조합 카운터 + OOS 강제(끌 수 없음) + 사전규칙 1종 P1 + 중립 렌더.
- **재무 팩터 랭킹 = P1 금지(배너로도 불가)**: 상폐사 재무 13.9%만 보유 → 분모 생존자 편향, G6 미통과(상폐사 재무 미수집이라 held-out 불가). 단일종목 재무 *게이트*(§4.10)는 허용. **칼 = 랭킹 금지 / 게이트 허용.**

### 2.9 ★유니버스 floor 근사 정직 (월말 prebuilt vs 일별 정밀)
- floor NAV에 *(근사·월말)* 접미 강제. "월중 손절·갭·정지 미반영 — 일별 정밀(local)과 다를 수 있음" 상존.
- floor는 월중 MDD 과소평가(월말끼리만) → "실제 낙폭 더 깊을 수 있음" 라벨. 월말효과·리밸일 운 포함 라벨.
- env.kind 분기로 "근사" 라벨 증발 금지(§4 capability presence). local에서 괴리 오버레이 비교 줄.

## 4. 로컬 bonus 경계

- **퍼블릭 floor**: P1-P3(다전략·정밀·리밸런싱) + P4 민감도/MC 거래재배열(서술·곡선). 전부 브라우저 TS 동기.
- **로컬 bonus**: walk-forward(재학습)·CPCV·DSR/PBO 수치(게이트 통과 시). floor를 대체 않고 보강. `env.kind` 분기로 floor에서 사라지지 않음 — capability presence로 분기(Python API 있으면 강건성 탭 노출).

## 5. "눈으로 확인" vs "눈으로 과적합" — 선의 정의

| 정직(눈으로 확인) | 무효(눈으로 과적합) |
|---|---|
| 되감아 t시점에 t까지만 보고 *한 번* 결정 | 전체차트 보며 시점결정 조정 / 되감아 수정 |
| 동일가중(사전규칙) 조합 | 곡선 보고 가중치 슬라이더 튜닝 |
| N전략 중립 비교 + OOS + selection 경고 | 최고 곡선 강조→선택(in-sample 미인대회) |
| 미래 = 가정 밴드 + 튜닝 출처 라벨 | 미래 = 검증된 단일 path, 경고 증발 |
| 단일종목 신호 규칙 차이 *서술* | 단일종목 "포트폴리오 분산효과" 주장 |

**선**: 결정 시점에 *쓰인 정보가 그 시점까지로 절단*되고 *사후 수정 불가*면 정직. 둘 중 하나라도 깨지면 무효.
