# 00 — 제품 가치 + 온보딩 페다고지

## 1. 진짜 가치 = 발견성, "온보딩 극장" 아님

레드팀 판정: 코어는 진짜 가치가 있으나 "투어"라는 단어가 위험하다. 진짜 가치는 *온보딩 의식(ritual)*이 아니라 **묻힌 자산의 발견성(discoverability)**이다.

- **지도(map)**: 투어가 이미 산다(606줄). 첫방문 1회 가치가 압도적이고 재실행은 거의 0 — 설계가 그걸 인정한다(`dartlab.map.tour.done` 후 자동시작 안 함). 사용자가 "map 들어가면 화면 보이며 안내한다"고 *좋은 예시로* 든 게 바로 이 동작이다.
- **터미널(terminal)**: 투어 0건(`grep tour|Tour|튜토리얼|onboard` → `terminal/` 0). 그런데 map보다 밀도가 높고 기능이 묻혀 있다 — 전체화면 차트, 심층 재무 탭, 스캔등급 다이얼로그, 섹션. **진짜 사용자 = "이 화면에 뭐가 있는지 몰라서 3개만 쓰는 신규/재방문자."** 이건 실수요다.

### 레드팀 직격: 투어 vs UI 단순화

"터미널이 복잡하니 투어로 가르치자"는 *위험한 대체재 회피*다. 13스텝 투어가 필요하다는 것 자체가 "화면이 너무 복잡하다"는 신호일 수 있다. 그러나 terminal-improvement PRD가 이미 "수직 분석은 초과 수준, 손대지 않는다"로 못박았으므로 — 단순화는 *별 트랙*이고, 묻힌 기능의 **발견성**은 독립적으로 실수요다.

> **명문 가드**: 투어는 발견성이지 단순화의 *대체*가 아니다. **7스텝으로 못 가르치면 그건 투어 결함이 아니라 UI 결함 신호다.** 투어가 복잡성의 핑계가 돼 단순화를 영영 미루면 안 된다.

## 2. 최소가치코어 (MVC)

> **터미널 헤더 `?` 버튼 하나 → map과 동일한 spotlight 엔진을 재사용한 짧은(5~7스텝) 비파괴 섹션 투어 + "이 기능 어디 있나"를 가리키는 것** — 자동생성·CHANGELOG파싱·게임화·텔레메트리·별도 패널 전부 없이.

이 코어가 서면 나머지(첫방문 코치마크·신규기능 안내·로컬/퍼블릭 주석)는 *직교 보완*이지 코어 의존이 아니다.

## 3. 청중 분리 — 투어(신규) vs 신규기능(재방문)

핵심 통찰: **투어 = "이게 뭔지 모르는 신규 사용자"**, **신규기능 = "이미 아는데 새 게 뭔지 궁금한 재방문 사용자"**. 두 청중이 정반대다. 하나의 UI에 욱여넣으면 둘 다 망한다 — 투어 첫 스텝에 "신규: X 추가!"를 넣으면 신규자는 X가 뭔지도 모르고, 재방문자는 전체 투어를 다시 봐야 새 걸 찾는다.

**설계: 헤더 버튼 1개 + 메뉴 분기 (헤더 청결 유지)**

```
[ 🎓 가이드 ▾ ]   ← 헤더 hdrLinks 군집에 버튼 1개 (unseen 신규기능 시 ✨N 뱃지)
   ├─ 📍 둘러보기 시작          (신규 사용자 → 투어)
   ├─ ✨ 새로워진 기능 (N)       (재방문자 → whatsNew, N=안 본 항목)
   └─ ⌨️ 단축키 보기            (보너스, 거의 무비용)
```

버튼 1개로 헤더 청결(덕지덕지 금지), 청중별 경로 분기. 신규기능 상세는 [03](03-whatsnew-feature-announcement.md).

## 4. 페다고지 — 깊이는 스텝 수가 아니라 트랙 분기로

map 투어의 영리한 설계는 **퀵 3 + 풀 13의 2층 구조**(`QUICK_COUNT=3`, `fullMode` boolean)다. 터미널은 패널이 14개+로 더 복잡하므로 스텝을 13→25로 늘리면 **아무도 안 끝낸다**(완주율 붕괴). 깊이는 **트랙 분기**로 푼다.

**터미널 = 퀵 4(선형) + 챕터 메뉴(사용자 선택)**

```
퀵투어 (4스텝, 강제 선형 — 첫 경험 상한 엄수):
  1. 환영 + "터미널은 한 회사를 360° 보는 작업대"
  2. ⌘K / GO — 회사를 어떻게 부르나 (가장 먼저 막히는 지점)
  3. 중앙 스택 = "이 회사 좋은가?" (스캔등급·종합판정·재무분석)
  4. 끝 + "더 깊이 볼 챕터를 고르세요" → 챕터 메뉴

풀투어 = 선형 강제 아님, 챕터 카드 (원하는 깊이만 선택):
  📊 재무 읽는 법   (재무분석·DuPont·재무제표·밸류에이션)
  ⚠️ 리스크·신용    (리스크·종합판정·DART팩트)
  🏭 업종 맥락       (업종백분위·스크리너·섹터히트맵·마켓펄스)
  🤖 AI·협업 도구    (AI·토론·이슈 — 로컬/퍼블릭 차이 여기서, [02] 참조)
```

map의 `fullMode` 2분기를 `track: string | null`로 *일반화*하면 최소 변경으로 챕터화된다(엔진 설계는 [01](01-shared-tour-engine-architecture.md) §1.2). 진행적 공개(progressive disclosure) 교과서 — 한 번에 다 보여주지 않고 사용자가 원하는 깊이만 고른다.

## 5. cardGuide 39개는 "가리키기만" — 백과사전화 금지

순진한 접근: "cardGuide 39개를 투어 스텝으로." **절대 금지** — 39스텝 투어는 누구도 안 본다. `cardGuide.ts`는 *이미* MiniFinChart `!` 툴팁이 소비하는 **맥락 적시 도움말(just-in-time help)**이다. 사용자가 그 카드를 볼 때 그 해석이 뜬다 — 이게 진짜 페다고지(맥락 적시 학습 > 사전 일괄 주입).

**투어의 역할 = "cardGuide가 있다는 걸 알려주는 것" 1스텝.** 재무 챕터에서:

> "각 차트 우측 상단 `!` 버튼을 누르면 — 이 지표가 뭔지, 좋은 신호/나쁜 신호가 한국어로 뜹니다. 39개 재무 카드 전부에. 외울 필요 없어요. (지금 눌러보세요 ▶)"

데모 콜백으로 실제 `!` 툴팁을 한 번 연다. 1스텝으로 39개 자산 전체를 활성화하는 레버리지. **투어 = 자산으로 가는 지도, 투어 ≠ 백과사전.** cardGuide를 투어가 중복 소비하면 SSOT 2개(유지보수 지옥) — 투어는 *가리키기만* 한다.

## 6. 큰 글씨 / 유도 카피 구체 스펙

map 투어 CSS 실측: `h3` 22px, `.body` 16px/1.75, popover 640px. 사용자 "글자 좀 크게" → **터미널은 한 단계 더 크게**:

| 요소 | map 현행 | 터미널 권장 | 근거 |
|---|---|---|---|
| 제목 h3 | 22px | **24px** | 터미널은 데이터 밀도 높아 투어 텍스트가 대비로 커야 읽힘 |
| 본문 body | 16px/1.75 | **17px/1.8** | 사용자 "글자 크게" 명시 |
| why 블록 | 14px | **15px** | 가치 카피는 작아도 됨 (보조) |
| 데모 버튼 | 13px | **15px, 풀폭 유지** | CTA는 커야 눌림. 현행 13px는 작다 |
| 팝오버 폭 | 640px | **680px** | 터미널 텍스트 길이 |

**유도 카피 원칙 (표시 원칙 준수)**:
- ✅ 동사·결과 중심: "여기를 누르면 화면 색이 바뀝니다"(행동→결과 즉시 보상). map이 이미 이 톤("화면 전체 색이 바뀌는 게 보입니다").
- ✅ 일인칭 질문 프레이밍: "이 회사 좋은가?에 3초 만에 답합니다"(map `why` 블록 계승).
- ✅ 데모 라벨 = `▶ [동사][결과]` 패턴(map 현행 "▶ ROE로 바꿔보기 (화면 색 변화)"). 괄호 안 결과 예고가 클릭률을 올린다.
- ⛔ 완결성·과장 금지: "모든 기능을 다 봤습니다"·"이제 전문가입니다" 금지. map 마지막 스텝이 본보기 — "툴은 만들었고, 인사이트는 당신이 찾아내는 겁니다"(겸손·절제).
- ⛔ "유도" ≠ 다크패턴: 닫기 숨김·"지금 안 보면 못 봄" 압박·자동 재팝업·강제 완주 금지. map처럼 Esc/건너뛰기/여기서 끝내기를 *항상* 노출(탈출 자유 패리티).

## 7. 첫방문 발견성 — 자동 모달 금지, 코치마크 허용

사용자는 "항상 on 금지·헤더 버튼"을 명시했다. 그러나 헤더 버튼은 *이미 길을 아는 사람만 누른다*(발견성 역설). 첫방문자를 어떻게 유도하나?

**절충: "자동 모달 금지, 1회성 코치마크는 허용"** (Phase 1)

- 첫 방문 시 화면을 덮는 투어는 **안 띄운다**(사용자 지시 준수). 대신 헤더 `?` 버튼에 **펄스 링 + 작은 말풍선 1개**만: "처음이세요? 여기서 1분 둘러보기 →". 클릭하면 투어, X·5초 경과·아무 패널 클릭 시 자동 소멸.
- 이건 "투어 자동 시작"이 아니라 "버튼을 가리키는 화살표"다. 화면 점유 0, 첫 탐색 차단 0, 버튼 발견성 해결. 모달 강제(역효과)와 영영 못 만남(발견성 0) 사이의 유일한 정답.
- localStorage는 `dartlab.{surface}.tour.done`(투어 완료)과 코치마크 노출 여부를 **분리**해야 한다(현 map은 done 하나뿐이라 "투어 안 봤지만 말풍선은 봤다"를 표현 못 함). 키 예산은 [05](05-scope-phasing-guardrails.md) §4.

> **지도(map) 기존 동작**: map은 이미 첫방문 자동시작(`!focus && !cmp` 가드로 딥링크 진입 시 억제). 사용자가 이걸 *좋은 예시*로 들었으므로 map의 첫방문 동작은 **유지**한다 — 단 `?` 재실행 버튼이 메뉴 분기(§3)로 승격된다. 터미널은 딥링크(`?sym=`) 진입이 많아(블로그·SNS 임베드) 자동시작이 *맥락 침입*이 되므로 Phase 0은 버튼만, 코치마크는 Phase 1.

## 8. 적대검증 — 이 기능이 망하는 5가지 + 방어

1. **짜증나는 투어(가장 흔한 사인死因)** — 14패널 선형 강제. 5번째에서 전원 Esc. → 방어: 챕터 분기로 선형 제거(§4)·Esc/건너뛰기 항상 노출·**퀵 4스텝 상한 엄수**·데모는 *되돌릴 수 있게*(map `clearSelectionAction` 패턴, 안 되돌리면 워크스페이스를 어지른 채 끝남).
2. **안 읽힘(텍스트 벽)** — map 8번 스텝 본문 6줄 "①~⑥"이 읽히는 길이 상한. → 방어: 스텝당 본문 3단락·각 2줄·나열 최대 6항목. 데모 버튼이 "읽기"를 "보기"로 옮김 — 텍스트는 보조, 화면 변화가 본체.
3. **유지보수 지옥(selector 깨짐)** — 가장 치명적. 패널 className 바꾸면 투어가 *조용히* 깨짐. → 방어: `data-tour` 안정 anchor·dev 가드·타입 union. 상세 [01](01-shared-tour-engine-architecture.md) §2.
4. **첫방문 강제의 역효과** — 사용자 명시 거부. → 방어: §7 코치마크. 자동 모달 절대 금지.
5. **로컬/퍼블릭 거짓말** — 퍼블릭에서 "AI 버튼 누르세요" 했는데 AI는 로컬 전용 → 없는 버튼을 가리킴. → 방어: `scope`/`env.kind` 필터, 상세 [02](02-local-public-common-wiring.md).

## 핵심 주장 3줄

1. **재발명 금지·일반화**: `TutorialTour.svelte`(606줄)+`cardGuide.ts`(39카드 just-in-time) 둘 다 프로덕션급 — 신규는 "터미널 콘텐츠 + data-tour anchor + 런타임 인지" 셋뿐, 셸은 공유 승격.
2. **깊이는 트랙 분기, 청중은 투어/신규기능 분리**: 터미널은 퀵4+챕터, cardGuide는 *가리키기만*, 신규기능은 `whatsNew.ts`가 `tourTrack`으로 투어 챕터 재사용(SSOT 1개).
3. **첫방문 강제 대신 코치마크, 완결성 대신 침묵**: 자동 모달은 거부됐으니 펄스 말풍선 1회로 발견성만 해결하고, "다 봤음"은 절대 단언 않으며(뱃지 숨김), 로컬/퍼블릭 차이는 *없는 버튼을 가리키지 않고* 있는 그대로 설명한다.
