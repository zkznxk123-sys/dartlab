# 유튜브 커뮤니티 게시물 파이프라인

> 유튜브 커뮤니티 탭용 이미지 게시물. 인스타 캐러셀(`INSTAGRAM.md`)과 자산·톤을 공유하되, 유튜브 사양에 맞게 크기·슬라이드 수·CTA를 조정한다.

---

## 1. 왜 유튜브 게시물인가

- 인스타그램 캐러셀은 노출이 팔로워 기반. 유튜브 커뮤니티는 **구독자 + 알고리즘 추천** 모두 작동.
- 유튜브 영상이 있는 기업이야기는 **영상 링크 + 이미지 게시물** 조합으로 영상 노출을 밀어올린다.
- 유튜브 게시물은 구글 검색에도 노출될 수 있다.

---

## 2. 사양

| 항목 | 유튜브 커뮤니티 | 인스타 캐러셀 (참고) |
|------|--------------|------------------|
| **이미지 크기** | **1080×1080 (1:1)** | 1080×1350 (4:5) |
| **대체 크기** | 1920×1080 (16:9) — 와이드형 | — |
| **이미지 수** | **1~10장** (갤러리) | 6~8장 |
| **GIF** | **지원** (자동 재생, 무음) | 미지원 |
| **텍스트** | 게시물 본문 500자+ 가능 | 캡션 2,200자 |
| **링크** | 본문에 URL 삽입 가능 | 링크 불가 (프로필만) |
| **CTA** | 블로그/영상 링크 직접 삽입 | "프로필 링크" 유도 |

### 이미지 크기 결정 기준

- **기업이야기 (차트·비교 중심)**: **1080×1080 (1:1)** — 모바일에서 정방형이 가장 크게 보인다
- **풍경·사진 중심**: **1920×1080 (16:9)** — 영상 썸네일과 동일한 느낌
- 같은 게시물에 1:1과 16:9 혼용 금지 — 한 게시물 내 크기 통일

---

## 3. 슬라이드 구성 — 이야기가 요구하는 만큼

유튜브 커뮤니티 갤러리는 **최대 10장**. 하지만 장수를 채우기 위해 내용을 쪼개지 않는다. **이야기가 요구하는 장수만 쓴다.**

### 기본 원칙

- **4~6장이 표준.** 대부분의 기업이야기는 4~6장이면 충분하다.
- 내용이 풍부하면 7~10장까지 자연스럽게 늘어날 수 있다.
- **1장짜리 텍스트 슬라이드를 만들기 위해 쪼개는 것 금지** — 한 장에 넣을 수 있으면 한 장에 넣는다.
- 캐러셀(6~8장)과 장수가 같아도 괜찮다. 크기(1:1 vs 4:5)와 CTA 구조가 다르므로 별개 콘텐츠다.

### 기업이야기 기본 구조 (4~6장)

| 장 | 역할 | 내용 |
|---|------|------|
| 1 | **후크** | 핵심 질문 + 배경 이미지 |
| 2 | **발견** | 핵심 숫자 + 차트 |
| 3 | **반전** | "그런데" 포인트 |
| 4 | **메커니즘** (선택) | 왜 그런지 구조 설명 |
| 5 | **비교** (선택) | 경쟁사·업종 대비 |
| 6 | **판단 + CTA** | 결론 + 영상/블로그 링크 |

### 3장 축약형 (빠른 게시물)

| 장 | 역할 |
|---|------|
| 1 | 후크 + 핵심 숫자 |
| 2 | 반전 + 메커니즘 |
| 3 | 판단 + CTA |

## 4. 비주얼 규칙 — 캐러셀과 공유하되 크기만 다르게

### CAROUSEL_DESIGN.md 상속 (그대로 적용)

- **3-Limit**: 컬러 3개(`#0E1116` + `#E8EAED` + `#6366F1`), 폰트 3종(Pretendard), 레이아웃 3종
- 이미지 베이스 필수 (검정 배경 금지)
- opacity 체계 (Hook 0.85, Chart 0.5)
- 정보밀도 60~80자/슬라이드
- 차트: 선그래프(성장), 막대(카테고리 비교)
- 이모지/스티커 금지

### 유튜브 전용 조정

| 항목 | 캐러셀 (4:5) | 유튜브 게시물 (1:1) |
|------|------------|------------------|
| 캔버스 | 1080×1350 | **1080×1080** |
| 헤드라인 최소 | 60px | **64px** (정방형이라 약간 키움) |
| 본문 최소 | 28px | **32px** |
| 차트 수치 | 44px | **48px** |
| 하단 여백 | 120px (캡션 공간) | **60px** (유튜브는 캡션 분리) |

### [최우선] 공간 활용 — 빈 곳 없이 채운다

**1080×1080 정방형은 면적이 작다. 빈 공간을 허용하지 않는다.**

나쁜 예: 상단 50%가 그라데이션만 깔려있고 텍스트가 하단에 몰림 → 정보 밀도 낮음, 스크롤에서 눈에 안 띔.

좋은 예: 전체 캔버스를 3~4 영역으로 나눠 각각 정보가 있음.

**레이아웃 원칙**:

1. **배경 이미지가 있으면** — 캔버스 전체를 덮되, 텍스트가 올라갈 영역에 그라데이션 오버레이. 상단에 주제/소제목, 중앙에 핵심 숫자, 하단에 출처/CTA. **세로 3등분에 각각 정보가 있어야 한다.**

2. **차트/표 슬라이드** — 차트가 캔버스의 60% 이상 차지. 나머지에 제목+한 줄 해석. 차트 위아래 여백 40px 이하.

3. **코드 슬라이드** (dartlab 시연) — 코드 블록이 캔버스의 50%+, 실행 결과(표/숫자)가 나머지. 코드 폰트 24px+로 모바일에서 읽혀야 함.

4. **비교 슬라이드** — 좌우 분할 또는 상하 분할로 2개 대상을 나란히. 빈 공간 없이 각 영역에 숫자+라벨.

**공간 체크**: 완성 후 각 슬라이드를 4등분(좌상/우상/좌하/우하)했을 때, **4영역 중 3개 이상에 정보(텍스트·숫자·차트·이미지)가 있어야 한다.** 2개 이하면 레이아웃 재배치.

### 아바타 활용 — 슬라이드 역할별 배치

dartlab 아바타(`landing/static/avatar-*.png`)를 슬라이드 우하단에 배치한다. **역할별 아바타가 고정**이므로 매 게시물 일관성 유지.

| 슬라이드 역할 | 아바타 | 표정/동작 |
|---|---|---|
| **후크** (질문) | `avatar-curious` | 물음표, 돋보기, 책 쌓아놓고 궁금해하는 |
| **발견** (차트/숫자) | `avatar-chart` | 차트 들고 분석하는 |
| **반전** (숨겨진 구조) | `avatar-detective` | 탐정 모자, 조사하는 |
| **메커니즘** (구조 설명) | `avatar-analyze` | 분석 중 |
| **비교** (경쟁사 대비) | `avatar-verify` | 검증하는 |
| **CTA** (코드/링크) | `avatar-code` | 코드 작성 중 |
| **판단** (결론) | `avatar-discover` | 발견한 표정 |

**배치 규칙**:
- 크기: **100~120px** (1080×1080 대비 약 10%)
- 위치: **우하단** (W-140, H-140) 고정
- 배경과 겹치지 않게 — dartlab 로고 텍스트 왼쪽으로 밀기
- GIF에서도 매 프레임 동일 위치에 고정

**render.py 구현**:
```python
AVATARS = {
    "hook": "avatar-curious.png",
    "discovery": "avatar-chart.png",
    "twist": "avatar-detective.png",
    "cta": "avatar-code.png",
}

def load_avatar(role, size=120):
    path = AVATAR_DIR / AVATARS[role]
    av = Image.open(path).convert("RGBA")
    return av.resize((size, size), Image.LANCZOS)
```

### 마지막 장 CTA 규칙

캐러셀 CTA는 "프로필 링크"로 유도하지만, 유튜브는 **본문에 링크를 직접 넣을 수 있다.**

- 마지막 장에 "자세한 분석은 영상에서" 또는 "블로그 전문 읽기" 텍스트
- 본문(텍스트)에 URL 삽입: `https://youtu.be/{videoId}` 또는 블로그 URL
- "구독" "좋아요" CTA는 마지막 장에 넣지 않는다 — 유튜브가 자체 버튼 제공

## 5. 자산 공유 — 블로그·캐러셀과 동일 풀

### 자산 경로 (ASSETS.md 규칙 그대로)

```
sns/assets/{stockCode-slug}/    ← 이미지 SSOT
  bg-hook-industrial.webp       ← 배경
  chart-opm-9y.webp             ← 차트 캡처
  photo-factory.webp            ← 실물 사진
  index.json                    ← 메타데이터
```

### 기업이야기 공동자산 활용 우선순위

1. **블로그 FLUX 이미지** (`blog/05-company-reports/{folder}/assets/*.webp`) — 이미 있으면 재사용
2. **sns/assets/ 공유 풀** — 캐러셀용으로 이미 만든 것 재사용
3. **신규 생성** — 위 두 곳에 없을 때만 Flux 호출 (Replicate `flux-1.1-pro` 전용)

### 블로그 자산 → 유튜브 게시물 변환

블로그 FLUX 이미지는 16:9 비율이다. 1:1로 변환할 때:

```python
# 16:9 → 1:1 센터 크롭
from PIL import Image
img = Image.open("blog-image-16x9.webp")
w, h = img.size
size = min(w, h)
left = (w - size) // 2
img_square = img.crop((left, 0, left + size, h))
img_square = img_square.resize((1080, 1080), Image.LANCZOS)
```

16:9 와이드형 게시물이면 크롭 없이 1920×1080으로 리사이즈.

## 6. 텍스트 (게시물 본문)

유튜브 커뮤니티 본문은 **500자 이상** 작성 가능. 인스타 캡션보다 자유롭다.

### 구조

```
[후크 한 줄 — 핵심 질문이나 놀라운 숫자]

[2~3문장 맥락 설명]

[핵심 발견 1~2개]

[CTA]
영상 전체 보기: https://youtu.be/{videoId}
블로그 전문: https://eddmpython.github.io/dartlab/blog/{slug}

#dartlab #기업분석 #재무제표 #{회사명}
```

### 톤 규칙 (INSTAGRAM.md 공통)

- 발견 톤 (설명 톤 아님)
- 숫자 자랑 금지
- 부채비율 과장 금지
- "무료" 표현 금지
- 존댓말 ("~습니다") — 유튜브 커뮤니티는 댓글 소통이 있으므로

## 7. 제작 파이프라인

### Phase 1: 소재 선정

- **영상 있는 기업이야기** → 영상 링크 + 핵심 장면 4장
- **블로그만 있는 기업이야기** → 블로그 링크 + 핵심 숫자 4장
- **독립 게시물** (설문·퀴즈) → 1장 이미지 + 텍스트

### Phase 2: 이미지 제작

1. 블로그/캐러셀 기존 자산 확인 (`blog/.../assets/`, `sns/assets/`)
2. 있으면 → 1:1 크롭 + 텍스트 오버레이
3. 없으면 → Flux 신규 생성 → `sns/assets/{code-slug}/` 저장
4. CAROUSEL_DESIGN.md 비주얼 규칙 적용 (크기만 1080×1080)

### Phase 3: 텍스트 작성

- 후크 한 줄 + 맥락 2~3문장 + CTA
- 해시태그 5~8개

### Phase 4: 게시

- 유튜브 스튜디오 → 커뮤니티 → 이미지 게시물
- 이미지 1~5장 업로드 + 본문 붙여넣기
- 영상 링크 있으면 본문에 삽입

## 9. 캐러셀 → 유튜브 게시물 변환 (같은 소재 재활용)

같은 기업이야기로 캐러셀(인스타)과 유튜브 게시물을 동시에 만들 때:

| 항목 | 인스타 캐러셀 | 유튜브 게시물 |
|------|------------|------------|
| 장수 | 6~8장 | **4~6장** (이야기가 요구하는 만큼) |
| 크기 | 1080×1350 | **1080×1080** |
| CTA | "프로필 링크" | **영상/블로그 URL 직접** |
| 캡션 | 존댓말, 긴 설명 | 존댓말, **짧은 핵심 + URL** |
| 배경 이미지 | **동일** (공유 풀) | **동일** |
| 차트/텍스트 | **동일 데이터** | **동일, 크기만 조정** |

변환 순서:
1. 캐러셀 6~8장 중 핵심 4장 선별 (후크/발견/반전/판단)
2. 4:5 → 1:1 리사이즈 (상하 크롭 또는 레이아웃 재배치)
3. 캡션 축약 + URL 추가

## 10. 렌더링 방식

이미지 생성에 3가지 방식을 활용한다. 소재 성격에 따라 선택.

### 방식 1: SVG → Pillow 합성 (기업이야기 차트/비교)

블로그 SVG 자산을 배경으로, Pillow로 텍스트 오버레이하여 1080×1080 webp 출력. `gen_thumbnails.py`와 같은 패턴.

```python
# SVG → PNG 변환 후 Pillow 합성
from PIL import Image, ImageDraw, ImageFont
# cairosvg.svg2png() 또는 Playwright screenshot
```

**적합**: 숫자 중심 차트, 비교표, 체크리스트

### 방식 2: HTML/CSS → Puppeteer/Playwright 캡처 (코드 시연·풍부한 레이아웃)

HTML 파일을 만들고 headless 브라우저로 1080×1080 스크린샷. 코드 하이라이팅, 테이블 렌더링, 복잡한 레이아웃에 적합.

```bash
# Playwright로 렌더
npx playwright screenshot --viewport-size="1080,1080" slide.html slide.webp
```

**적합**: dartlab 코드 시연, API 설명, 복잡한 레이아웃

### 방식 3: Remotion 컴포넌트 재활용 (캐러셀과 동일 소재)

기존 `remotion-sns/` 컴포넌트를 1080×1080으로 리사이즈하여 스틸 캡처. 캐러셀과 동일 소재를 유튜브용으로 변환할 때.

```bash
npx remotion still --props='{"width":1080,"height":1080}' SlideComponent output.webp
```

**적합**: 캐러셀 → 유튜브 변환, 동일 디자인 시스템 유지

### 방식 4: GIF 애니메이션 (코드 실행 과정·차트 변화)

유튜브 커뮤니티는 **GIF를 지원**한다. 피드에서 자동 재생(무음)되므로 스크롤 중 시선을 잡는 데 정적 이미지보다 강력하다.

**GIF 적합 소재**:
- **dartlab 코드 실행 과정**: 코드 입력 → 실행 → 결과 테이블 순차 등장 (3~5초)
- **차트 변화**: 9년 시계열이 연도별로 쌓이는 애니메이션 (2~4초)
- **비교 전환**: A회사 → B회사 숫자가 전환되는 효과 (2~3초)

**GIF 제작 방법**:

```python
# Pillow로 프레임 합성 → GIF
from PIL import Image
frames = [Image.open(f"frame_{i}.png") for i in range(10)]
```

```bash
# Playwright로 HTML 애니메이션 녹화 → GIF
```

**GIF 규칙**:
- 크기: **1080×1080**, 파일 **8MB 이하** (유튜브 제한)
- 길이: **2~6초**, 루프 재생
- fps: **10~15** (용량 절약, 부드러움 균형)
- 텍스트 최소 크기: **40px** (모바일 GIF는 작게 보임)
- GIF 1개 + 정적 이미지 조합 가능 (GIF를 1장째 후크로)

## 11. 금지 패턴

- 영상 썸네일을 그대로 1장만 올리는 게시물 — 정보가 없다
- 텍스트만 있는 게시물 — 이미지 없으면 노출 안 됨
- 캐러셀 10장을 그대로 올리는 것 — 유튜브 최대 5장, 압축 필수
- 16:9와 1:1 혼용 — 한 게시물 내 크기 통일
- "구독 좋아요" CTA를 이미지 안에 넣기 — 유튜브가 자체 제공
- 같은 이미지를 Flux로 다시 생성 — 공유 풀에서 가져오기
- **상단 50%가 비어있는 슬라이드** — 4등분 중 3영역 이상에 정보 필수
- 장수 채우기 위해 내용을 억지로 쪼개는 것 — 4장이면 4장, 7장이면 7장
