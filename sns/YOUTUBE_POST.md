# 유튜브 커뮤니티 게시물 파이프라인

> 유튜브 커뮤니티 탭용 이미지 게시물. 인스타 캐러셀(`INSTAGRAM.md`)과 자산·톤을 공유하되, 유튜브 사양에 맞게 크기·슬라이드 수·CTA 를 조정한다.

각 섹션은 **"이렇게 한다"** 명제로 열고, 반복된 실수는 섹션 하단 **"반복 실패"** 에 작게 정리한다.

---

## 1. 왜 유튜브 게시물인가 — 영상과 이미지 조합으로 양방향 밀어올린다

- 인스타그램 캐러셀은 노출이 팔로워 기반. 유튜브 커뮤니티는 **구독자 + 알고리즘 추천** 모두 작동.
- 유튜브 영상이 있는 기업이야기는 **영상 링크 + 이미지 게시물** 조합으로 영상 노출을 밀어올린다.
- 유튜브 게시물은 구글 검색에도 노출될 수 있다.

---

## 2. 사양 — 1080×1080 정방형, 1~10 장 갤러리로 간다

| 항목 | 유튜브 커뮤니티 | 인스타 캐러셀 (참고) |
|---|---|---|
| **이미지 크기** | **1080×1080 (1:1)** | 1080×1350 (4:5) |
| **대체 크기** | 1920×1080 (16:9) — 와이드형 | — |
| **이미지 수** | **1~10 장** (갤러리) | 6~8 장 |
| **GIF** | **지원** (자동 재생, 무음) | 미지원 |
| **텍스트** | 게시물 본문 500 자+ 가능 | 캡션 2,200 자 |
| **링크** | 본문에 URL 삽입 가능 | 링크 불가 (프로필만) |
| **CTA** | 블로그/영상 링크 직접 삽입 | "프로필 링크" 유도 |

### 이미지 크기 선택

- **기업이야기 (차트·비교 중심)** → **1080×1080 (1:1)**. 모바일에서 정방형이 가장 크게 보인다.
- **풍경·사진 중심** → **1920×1080 (16:9)**. 영상 썸네일과 동일한 느낌.
- 한 게시물 내 크기 통일.

**반복 실패** — 같은 게시물에 1:1 과 16:9 혼용 시 피드 그리드가 어긋나 프로 느낌을 해친다.

---

## 3. 슬라이드 — 이야기가 요구하는 4~6 장만 쓴다

유튜브 커뮤니티 갤러리는 최대 10 장이지만 장수를 채우기 위해 내용을 쪼개지 않는다. 이야기가 요구하는 장수만 쓴다.

### 기본 원칙

- **4~6 장이 표준.** 대부분의 기업이야기는 4~6 장이면 충분하다.
- 내용이 풍부하면 7~10 장까지 자연스럽게 늘어나도 된다.
- 한 장에 넣을 수 있으면 한 장에 넣는다.
- 캐러셀(6~8 장)과 장수가 같아도 OK — 크기·CTA 구조가 다르므로 별개 콘텐츠.

### 기업이야기 기본 구조 (4~6 장)

| 장 | 역할 | 내용 |
|---|---|---|
| 1 | 후크 | 핵심 질문 + 배경 이미지 |
| 2 | 발견 | 핵심 숫자 + 차트 |
| 3 | 반전 | "그런데" 포인트 |
| 4 | 메커니즘 (선택) | 왜 그런지 구조 설명 |
| 5 | 비교 (선택) | 경쟁사·업종 대비 |
| 6 | 판단 + CTA | 결론 + 영상/블로그 링크 |

### 3 장 축약형 (빠른 게시물)

| 장 | 역할 |
|---|---|
| 1 | 후크 + 핵심 숫자 |
| 2 | 반전 + 메커니즘 |
| 3 | 판단 + CTA |

**반복 실패** — 1 장짜리 텍스트 슬라이드를 만들기 위해 내용을 쪼개면 빈 슬라이드가 섞여 완주율 저하. 캐러셀 10 장을 그대로 가져오면 유튜브 사양(최대 10 장 허용이되 피드에서 5 장 이후 집중력 급락)에 맞지 않는다.

## 4. 비주얼 — 캐러셀 규칙 상속 + 1080×1080 조정

### CAROUSEL_DESIGN.md 상속 (그대로 적용)

- 3-Limit — 컬러 3 개 (`#0E1116` + `#E8EAED` + `#6366F1`), 폰트 3 종 (Pretendard), 레이아웃 3 종.
- 이미지 베이스 필수 (모든 씬에 배경 이미지).
- opacity 체계 (Hook 0.85 / Chart 0.5).
- 정보 밀도 60~80 자/슬라이드.
- 차트 — 선그래프(성장), 막대(카테고리 비교).

### 유튜브 전용 조정

| 항목 | 캐러셀 (4:5) | 유튜브 게시물 (1:1) |
|---|---|---|
| 캔버스 | 1080×1350 | **1080×1080** |
| 헤드라인 최소 | 60px | **64px** (정방형이라 약간 키움) |
| 본문 최소 | 28px | **32px** |
| 차트 수치 | 44px | **48px** |
| 하단 여백 | 120px (캡션 공간) | **60px** (유튜브는 캡션 분리) |

### 공간 활용 — 4 등분 중 3 영역 채운다

1080×1080 정방형은 면적이 작다. 빈 공간을 허용하지 않는다.

**레이아웃 원칙**:

1. **배경 이미지가 있으면** — 캔버스 전체를 덮되 텍스트 영역에 그라데이션 오버레이. 상단에 주제/소제목, 중앙에 핵심 숫자, 하단에 출처/CTA. **세로 3 등분에 각각 정보가 있다.**
2. **차트·표 슬라이드** — 차트가 캔버스의 60% 이상. 나머지에 제목 + 한 줄 해석. 차트 위아래 여백 40px 이하.
3. **코드 슬라이드** (dartlab 시연) — 코드 블록이 캔버스의 50%+, 실행 결과(표·숫자)가 나머지. 코드 폰트 24px+ 로 모바일에서 읽혀야 한다.
4. **비교 슬라이드** — 좌우 분할 또는 상하 분할로 2 개 대상을 나란히. 빈 공간 없이 각 영역에 숫자 + 라벨.

**공간 체크**: 완성 후 각 슬라이드를 4 등분(좌상·우상·좌하·우하) 했을 때 **4 영역 중 3 개 이상에 정보(텍스트·숫자·차트·이미지)가 있다.** 2 개 이하면 레이아웃 재배치.

**반복 실패** — 상단 50% 가 그라데이션만 깔려있고 텍스트가 하단에 몰림 → 정보 밀도 낮음, 스크롤에서 눈에 안 띔.

### 아바타 — 슬라이드 역할별 배치

dartlab 아바타(`landing/static/avatar-*.png`) 를 슬라이드 우하단에 배치. 역할별 아바타가 고정이라 매 게시물 일관성 유지.

| 슬라이드 역할 | 아바타 | 표정·동작 |
|---|---|---|
| **후크** (질문) | `avatar-curious` | 물음표, 돋보기, 책 쌓아놓고 궁금해하는 |
| **발견** (차트·숫자) | `avatar-chart` | 차트 들고 분석 |
| **반전** (숨겨진 구조) | `avatar-detective` | 탐정 모자, 조사 |
| **메커니즘** (구조 설명) | `avatar-analyze` | 분석 중 |
| **비교** (경쟁사 대비) | `avatar-verify` | 검증 |
| **CTA** (코드·링크) | `avatar-code` | 코드 작성 |
| **판단** (결론) | `avatar-discover` | 발견한 표정 |

**배치 규칙**:
- 크기 **100~120px** (1080×1080 대비 약 10%)
- 위치 **우하단** (W-140, H-140) 고정
- 배경과 겹치지 않게 — dartlab 로고 텍스트 왼쪽으로 밀기
- GIF 에서도 매 프레임 동일 위치에 고정

**render.py 구현**:
```python
AVATARS = {
    "hook": "avatar-curious.png",
    "discovery": "avatar-chart.png",
    "twist": "avatar-detective.png",
    "cta": "avatar-code.png",
}

def load_avatar(role, size=120):
    path = AVATAR_DIR / AVATARS[role]
    av = Image.open(path).convert("RGBA")
    return av.resize((size, size), Image.LANCZOS)
```

### 마지막 장 CTA

캐러셀 CTA 는 "프로필 링크" 로 유도하지만 유튜브는 본문에 링크를 직접 넣을 수 있다.

- 마지막 장에 "자세한 분석은 영상에서" 또는 "블로그 전문 읽기" 텍스트.
- 본문(텍스트) 에 URL 삽입: `https://youtu.be/{videoId}` 또는 블로그 URL.

**반복 실패** — "구독 · 좋아요" CTA 를 이미지 안에 넣으면 유튜브 자체 버튼과 중복. 이미지 안 CTA 대신 본문 링크 활용.

## 5. 자산 — 블로그·캐러셀 공유 풀에서 가져온다

### 자산 경로 (`ASSETS.md` 규칙)

```
sns/assets/{stockCode-slug}/    ← 이미지 SSOT
  bg-hook-industrial.webp       ← 배경
  chart-opm-9y.webp             ← 차트 캡처
  photo-factory.webp            ← 실물 사진
  index.json                    ← 메타데이터
```

### 공동자산 우선순위

1. **블로그 FLUX 이미지** (`blog/05-company-reports/{folder}/assets/*.webp`) — 이미 있으면 재사용
2. **sns/assets/ 공유 풀** — 캐러셀용으로 이미 만든 것 재사용
3. **신규 생성** — 위 두 곳에 없을 때만 Flux 호출 (Replicate `flux-1.1-pro`)

### 블로그 → 유튜브 변환 (16:9 → 1:1)

```python
from PIL import Image
img = Image.open("blog-image-16x9.webp")
w, h = img.size
size = min(w, h)
left = (w - size) // 2
img_square = img.crop((left, 0, left + size, h))
img_square = img_square.resize((1080, 1080), Image.LANCZOS)
```

16:9 와이드형 게시물이면 크롭 없이 1920×1080 으로 리사이즈.

**반복 실패** — 같은 이미지를 Flux 로 다시 생성. 공유 풀에서 먼저 찾는다.

## 6. 본문 — 후크·맥락·CTA·해시태그 순으로 쓴다

유튜브 커뮤니티 본문은 **500 자 이상** 작성 가능. 인스타 캡션보다 자유롭다.

### 구조

```
[후크 한 줄 — 핵심 질문이나 놀라운 숫자]

[2~3 문장 맥락 설명]

[핵심 발견 1~2 개]

[CTA]
영상 전체 보기: https://youtu.be/{videoId}
블로그 전문: https://eddmpython.github.io/dartlab/blog/{slug}

#dartlab #기업분석 #재무제표 #{회사명}
```

### 톤

- 발견 톤, 설명 톤 아님
- 존댓말 "~습니다" — 유튜브 커뮤니티는 댓글 소통이 있으므로

**반복 실패** — 숫자 자랑 ("14 축") / 부채비율 과장 / "무료" 표현 / "설명드리자면" 같은 관료체.

## 7. 파이프라인 — 4 단계로 굴린다

### Phase 1: 소재 선정
- 영상 있는 기업이야기 → 영상 링크 + 핵심 장면 4 장
- 블로그만 있는 기업이야기 → 블로그 링크 + 핵심 숫자 4 장
- 독립 게시물 (설문·퀴즈) → 1 장 이미지 + 텍스트

### Phase 2: 이미지 제작
1. 블로그/캐러셀 기존 자산 확인 (`blog/.../assets/`, `sns/assets/`)
2. 있으면 1:1 크롭 + 텍스트 오버레이
3. 없으면 Flux 신규 생성 → `sns/assets/{code-slug}/` 저장
4. CAROUSEL_DESIGN.md 비주얼 규칙 적용 (크기만 1080×1080)

### Phase 3: 텍스트 작성
- 후크 한 줄 + 맥락 2~3 문장 + CTA
- 해시태그 5~8 개

### Phase 4: 게시
- 유튜브 스튜디오 → 커뮤니티 → 이미지 게시물
- 이미지 1~5 장 업로드 + 본문 붙여넣기
- 영상 링크 있으면 본문에 삽입

## 9. 캐러셀 → 유튜브 변환 — 4:5 를 1:1 로 옮긴다

같은 기업이야기로 캐러셀(인스타)과 유튜브 게시물을 동시에 만들 때:

| 항목 | 인스타 캐러셀 | 유튜브 게시물 |
|---|---|---|
| 장수 | 6~8 장 | **4~6 장** (이야기가 요구하는 만큼) |
| 크기 | 1080×1350 | **1080×1080** |
| CTA | "프로필 링크" | **영상·블로그 URL 직접** |
| 캡션 | 존댓말, 긴 설명 | 존댓말, **짧은 핵심 + URL** |
| 배경 이미지 | 동일 (공유 풀) | 동일 |
| 차트·텍스트 | 동일 데이터 | 동일, 크기만 조정 |

**변환 순서**:
1. 캐러셀 6~8 장 중 핵심 4 장 선별 (후크·발견·반전·판단)
2. 4:5 → 1:1 리사이즈 (상하 크롭 또는 레이아웃 재배치)
3. 캡션 축약 + URL 추가

## 10. 렌더링 — 4 방식 중 소재에 맞는 것을 쓴다

이미지 생성에 4 가지 방식을 활용한다. 소재 성격에 따라 선택.

### 방식 1: SVG → Pillow 합성 (기업이야기 차트·비교)

블로그 SVG 자산을 배경으로, Pillow 로 텍스트 오버레이하여 1080×1080 webp 출력. `gen_thumbnails.py` 와 같은 패턴.

```python
from PIL import Image, ImageDraw, ImageFont
# cairosvg.svg2png() 또는 Playwright screenshot
```

**적합**: 숫자 중심 차트, 비교표, 체크리스트.

### 방식 2: HTML/CSS → Puppeteer/Playwright (코드 시연·풍부한 레이아웃)

HTML 파일을 만들고 headless 브라우저로 1080×1080 스크린샷. 코드 하이라이팅·테이블·복잡한 레이아웃에 적합.

```bash
npx playwright screenshot --viewport-size="1080,1080" slide.html slide.webp
```

**적합**: dartlab 코드 시연, API 설명, 복잡한 레이아웃.

### 방식 3: Remotion 컴포넌트 재활용 (캐러셀과 동일 소재)

기존 `remotion-sns/` 컴포넌트를 1080×1080 으로 리사이즈하여 스틸 캡처. 캐러셀과 동일 소재를 유튜브용으로 변환할 때.

```bash
npx remotion still --props='{"width":1080,"height":1080}' SlideComponent output.webp
```

**적합**: 캐러셀 → 유튜브 변환, 동일 디자인 시스템 유지.

### 방식 4: GIF 애니메이션 (코드 실행·차트 변화)

유튜브 커뮤니티는 GIF 를 지원한다. 피드에서 자동 재생(무음)되므로 스크롤 중 시선을 잡는 데 정적 이미지보다 강력하다.

**적합 소재**:
- dartlab 코드 실행 과정 (코드 입력 → 실행 → 결과 테이블 순차 등장, 3~5 초)
- 차트 변화 (9 년 시계열이 연도별로 쌓이는 애니메이션, 2~4 초)
- 비교 전환 (A 회사 → B 회사 숫자가 전환, 2~3 초)

**제작 방법**:

```python
# Pillow 프레임 합성 → GIF
from PIL import Image
frames = [Image.open(f"frame_{i}.png") for i in range(10)]
frames[0].save("output.gif", save_all=True, append_images=frames[1:],
               duration=500, loop=0, optimize=True)
```

```bash
# Playwright HTML 애니메이션 녹화 → GIF
# 또는 Remotion → MP4 → ffmpeg GIF 변환
ffmpeg -i animation.mp4 -vf "fps=10,scale=1080:1080" -loop 0 output.gif
```

**GIF 규칙**:
- 크기 **1080×1080**, 파일 **8MB 이하** (유튜브 제한)
- 길이 **2~6 초**, 루프 재생
- fps **10~15** (용량 절약, 부드러움 균형)
- 텍스트 최소 크기 **40px** (모바일 GIF 는 작게 보임)
- GIF 1 개 + 정적 이미지 조합 가능 (GIF 를 1 장째 후크로)

### 유형별 추천

| 유형 | 렌더링 방식 | 이유 |
|---|---|---|
| 기업이야기 (차트) | SVG+Pillow 또는 HTML | 블로그 SVG 재활용 |
| dartlab 코드 시연 | **HTML/CSS+Playwright** | 코드 하이라이팅 필수 |
| 코드 실행 과정 | **GIF** (Pillow 프레임 합성) | 자동재생으로 시선 잡기 |
| 차트 시계열 변화 | **GIF** (프레임별 연도 추가) | 정적 차트보다 임팩트 |
| 캐러셀 변환 | Remotion still | 디자인 일관성 |
| 사진 중심 | Pillow (FLUX 배경+텍스트) | 단순 합성 |

**반복 실패** — 영상 썸네일을 그대로 1 장만 올리는 게시물 (정보 없음) / 텍스트만 있는 게시물 (이미지 없으면 노출 안 됨).

## 요약 — 명제 10 줄

1. 영상과 이미지 조합으로 유튜브 알고리즘을 양방향으로 밀어올린다.
2. 사양은 1080×1080 정방형 기본, 1~10 장 갤러리.
3. 장수는 이야기가 요구하는 4~6 장만 쓴다.
4. 비주얼은 캐러셀 규칙 상속 + 정방형 크기 조정.
5. 4 등분 중 3 영역 이상 정보로 채워 공간 낭비 없이.
6. 아바타를 슬라이드 역할별로 우하단 120px 고정 배치.
7. 자산은 블로그·캐러셀 공유 풀에서 먼저 찾아 쓴다.
8. 본문은 후크 · 맥락 2~3 문장 · CTA · 해시태그 순.
9. 캐러셀 소재를 1:1 로 옮기며 핵심 4 장만 선별한다.
10. 렌더링은 SVG+Pillow · HTML+Playwright · Remotion still · GIF 4 방식 중 소재에 맞는 것으로.
